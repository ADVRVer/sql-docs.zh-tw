# <a name="user-interface-reference-oracletosql"></a>使用者介面參考 (OracleToSQL)
本節包含適用于 Oracle 的 [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] Migration Assistant (SSMA) 的說明主題。  
  
## <a name="in-this-section"></a>本節內容  
下表列出 SSMA 對話方塊：  
  
|主題|描述|  
|-|-|  
|[OracleToSQL&#41;的 Advanced Object Selection &#40;](../../ssma/oracle/advanced-object-selection-oracletosql.md)|使用 [ **Advanced Object 選取** ] 對話方塊，即可使用篩選準則來尋找資料庫物件，然後選取或清除這些物件。|  
|[評量報表 &#40;OracleToSQL&#41;](../../ssma/oracle/assessment-report-oracletosql.md)|您可以使用評量報告來查看將 Oracle 物件轉換成語法的結果 [!INCLUDE[tsql](../../includes/tsql-md.md)] ，以及估計遷移至的時間和複雜度 [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] 。|  
|[連接到 Oracle &#40;OracleToSQL&#41;](../../ssma/oracle/connect-to-oracle-oracletosql.md)|使用 [ **連接到 oracle** ] 對話方塊，即可連接到您想要遷移的 Oracle 資料庫。|  
|[連接至 SQL Server &#40;OracleToSQL&#41;](../../ssma/oracle/connect-to-sql-server-oracletosql.md)|使用 [ **連接到 SQL Server** ] 對話方塊，即可連接到 [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] 您要遷移至的實例。|  
|[資料移轉報表 &#40;OracleToSQL&#41;](../../ssma/oracle/data-migration-report-oracletosql.md)|顯示將資料從 Oracle 遷移至的結果 [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] 。|  
|[資料移轉設定](data-migration-settings-oracletosql.md)|您可以使用 [ **擴充資料移轉設定** ] 索引標籤來撰寫資料移轉的自訂查詢。|  
|[編輯類型對應 &#40;OracleToSQL&#41;](../../ssma/oracle/edit-type-mapping-oracletosql.md)|使用 [ **新增類型對應** ] 或 [ **編輯類型對應** ] 對話方塊，即可建立或修改來源和目標資料庫與資料庫物件之間的資料類型對應。|  
|[全域設定 &#40;編輯器&#41; &#40;OracleToSQL&#41;](../../ssma/oracle/global-settings-editor-oracletosql.md)|使用 [ **通用設定** ] 對話方塊的 [編輯器] 頁面，即可設定程式碼編輯器選項。|  
|[&#40;對話方塊的全域設定&#41;  &#40;OracleToSQL&#41;](../../ssma/oracle/global-settings-dialogs-oracletosql.md)|使用 [ **通用設定** ] 對話方塊的 [對話方塊] 頁面，即可設定預設對話方塊和警告設定。|  
|[全域設定 &#40;記錄&#41; &#40;OracleToSQL&#41;](../../ssma/oracle/global-settings-logging-oracletosql.md)|使用 [ **通用設定** ] 對話方塊的 [記錄] 頁面，即可設定記錄。|  
|[全域設定 &#40;輸出視窗&#41;  &#40;OracleToSQL&#41;](../../ssma/oracle/global-settings-output-window-oracletosql.md)|使用 [ **通用設定** ] 對話方塊，即可設定 SSMA for Oracle 使用者介面的喜好設定。|  
|[&#40;測試人員的全域設定&#41; &#40;OracleToSQL&#41;](../../ssma/oracle/global-settings-tester-oracletosql.md)|使用 [ **GlobalSettings** ] 對話方塊的 [測試人員] 頁面，即可指定測試人員的設定。|  
|[新專案 &#40;OracleToSQL&#41;](../../ssma/oracle/new-project-oracletosql.md)|使用 [ **新增專案** ] 對話方塊來建立 Oracle 專案的新 SSMA。|  
|[專案設定 &#40;轉換&#41; &#40;OracleToSQL&#41;](../../ssma/oracle/project-settings-conversion-oracletosql.md)|使用 [ **專案設定** ] 對話方塊的 [轉換] 頁面，即可指定 ORACLE 的 SSMA 轉換函數和全域變數的方式。|  
|[專案設定 &#40;GUI&#41; &#40;OracleToSQL&#41;](../../ssma/oracle/project-settings-gui-oracletosql.md)|使用 [ **專案設定** ] 對話方塊的 [GUI] 頁面，即可指定 [ **資料** ] 索引標籤上所顯示的資料量。|  
|[&#40;遷移&#41; &#40;OracleToSQL&#41;的專案設定 ](../../ssma/oracle/project-settings-migration-oracletosql.md)|您可以使用 [ **專案設定** ] 對話方塊的 [遷移] 頁面，自訂 ORACLE 的 SSMA 將資料從 oracle 遷移至的方式 [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] 。|  
|[&#40;同步處理的專案設定&#41; &#40;OracleToSQL&#41;](../../ssma/oracle/project-settings-synchronization-oracletosql.md)|您可以使用 [ **專案設定** ] 對話方塊的 [同步處理] 頁面，自訂 ORACLE 的 SSMA 在中建立或改變遷移之資料庫物件的方式 [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] 。|  
|[專案設定&#40;載入系統物件&#41; &#40;OracleToSQL&#41;](../../ssma/oracle/project-settings-loading-system-objects-oracletosql.md)|使用 [ **專案設定** ] 對話方塊的 [載入系統物件] 頁面，即可指定 SSMA 轉換和載入的 Oracle 系統物件 [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] 。|  
|[專案設定 &#40;類型對應&#41; &#40;OracleToSQL&#41;](../../ssma/oracle/project-settings-type-mapping-oracletosql.md)|使用 [ **專案設定** ] 對話方塊的 [類型對應] 頁面，即可指定 SSMA for Oracle 專案中所有資料庫和資料庫物件的預設型別對應。|  
|[從資料庫重新整理 &#40;OracleToSQL&#41;](../../ssma/oracle/refresh-from-database-oracletosql.md)|您可以使用 [ **從資料庫** 重新整理] 對話方塊來選取要從 Oracle 資料庫重新整理的物件。|  
|[將中繼資料儲存 &#40;OracleToSQL&#41;](../../ssma/oracle/save-metadata-oracletosql.md)|當您儲存缺少中繼資料的專案時，會出現 [ **儲存中繼資料** ] 對話方塊。|
