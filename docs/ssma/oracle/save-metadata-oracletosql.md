# <a name="save-metadata--oracletosql"></a>儲存中繼資料 (OracleToSQL)
[ **儲存中繼資料** ] 對話方塊會提示您將中繼資料載入至 SSMA 專案，然後儲存它。 這可讓您擁有完整的專案檔，讓您可以離線使用並傳送給其他人，例如技術支援人員。  
  
若要存取 [ **儲存中繼資料** ] 對話方塊，請儲存專案。 如果遺漏任何中繼資料，SSMA 會顯示 [ **儲存中繼資料** ] 對話方塊。  
  
## <a name="options"></a>選項。  
**名稱**  
專案中每個資料庫的名稱。  
  
**狀態**  
指出中繼資料是否載入至 SSMA 專案，或中繼資料是否遺失。  
  
SSMA 會視需要將中繼資料載入至專案。 當您流覽中繼資料並轉換架構時，會自動載入中繼資料。  
  
**全選**  
選取所有列出的資料庫。  
  
**Clear**  
清除所有遺失中繼資料之資料庫的核取方塊。 如果已載入中繼資料，您就無法清除該核取方塊。  
  
**儲存**  
儲存專案，載入具有遺漏中繼資料之所選資料庫的中繼資料。  
  
**取消**  
取消儲存作業。 遺漏中繼資料不會載入至專案。
