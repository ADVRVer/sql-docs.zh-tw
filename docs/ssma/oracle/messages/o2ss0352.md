# <a name="o2ss0352-bulk-collect-into-clause-in-select-statement-not-converted-error"></a>O2SS0352： SELECT 語句中的 BULK COLLECT INTO 子句未轉換（錯誤）

本文涵蓋為何當語句搭配`SELECT` `BULK COLLECT INTO`子句和`DISTINCT`子句使用時，Oracle 的 SQL Server 移轉小幫手（SSMA）不會轉換 PL/SQL 區塊。

## <a name="background"></a>背景

集合是專案的已排序群組，全都是相同的類型。 這是包含清單、陣列和其他熟悉資料類型的一般概念。 每個元素都有唯一的注標，以決定其在集合中的位置。 `DISTINCT`子句會指定只有唯一的值可以出現在結果集中。 `BULK COLLECT INTO`子句會將查詢的輸出系結至集合，進而減少 PL/SQL 和 SQL 引擎之間的通訊。

SSMA 提供 Oracle 集合的遷移路徑。 不過，如果用`SELECT`來填入集合的語句使用`DISTINCT`子句，則 SSMA 會產生錯誤訊息。

## <a name="example"></a>範例

請考慮下列搭配`SELECT` `DISTINCT`子句和`BULK COLLECT INTO`子句使用語句的範例：

```sql
CREATE OR REPLACE PROCEDURE BULK_COLLECT_WITH_DISTINCT
AS
BEGIN
    DECLARE
        TYPE tbl_type_DNAME IS TABLE OF varchar2(14);
        var_dname tbl_type_DNAME;
    BEGIN
        SELECT DISTINCT DNAME
        BULK COLLECT INTO var_dname
        FROM dept;

        IF var_dname.COUNT = 0 THEN
            DBMS_OUTPUT.PUT_LINE('No results!');
        ELSE
            DBMS_OUTPUT.PUT_LINE('Results:');

            FOR i IN var_dname.FIRST..var_dname.LAST LOOP
                DBMS_OUTPUT.PUT_LINE(' DEPT NAME #' || var_dname(i));
            END LOOP;
        END IF;
    END;
END;
```

當您嘗試在 SSMA 中轉換上述程式碼時，它會產生下列錯誤訊息：

> O2SS0352： SELECT 語句中的 BULK COLLECT INTO 子句未轉換

## <a name="possible-remedies"></a>可能的解決辦法

若要轉換上述的 PL/SQL 區塊，我們必須`DISTINCT`將子句`UNIQUE`取代為子句（的`DISTINCT`同義字），如下所示：

```sql
CREATE OR REPLACE PROCEDURE BULK_COLLECT_WITH_DISTINCT
AS
BEGIN
    DECLARE
        TYPE tbl_type_DNAME IS TABLE OF varchar2(14);
        var_dname tbl_type_DNAME;
    BEGIN
        SELECT UNIQUE DNAME
        BULK COLLECT INTO var_dname
        FROM dept;

        IF var_dname.COUNT = 0 THEN
            DBMS_OUTPUT.PUT_LINE('No results!');
        ELSE
            DBMS_OUTPUT.PUT_LINE('Results:');

            FOR i IN var_dname.FIRST..var_dname.LAST LOOP
                DBMS_OUTPUT.PUT_LINE(' DEPT NAME #' || var_dname(i));
            END LOOP;
        END IF;
    END;

END;
```
