本節包含 [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] 移轉小幫手 (SSMA) 以進行存取的說明主題。

|[尋找資料庫 Wizard (選取檔案) ](https://msdn.microsoft.com/2f574a34-4bab-40a4-89a8-ad4907ffc3fd)|使用 [尋找資料庫] 的 [選取檔案] 頁面，即可選取要加入 SSMA 專案的 Access 資料庫。|  
|[尋找資料庫 Wizard (選取位置) ](https://msdn.microsoft.com/00b2d32a-998b-47a7-b25c-589b5bd6777a)|使用 [尋找資料庫] Wizard 的 [選取位置] 頁面，即可指定要在網路、電腦或符合搜尋條件之資料夾中搜尋 Access 資料庫檔案的位置。|  
|[尋找資料庫精靈 (驗證選取項目)](https://msdn.microsoft.com/62e20e03-50cc-4ac8-8072-524d194d2ec3)|使用 [尋找資料庫] Wizard 的 [驗證選取範圍] 頁面，確認將加入至 SSMA 專案的資料庫。|

|[&#40;對話方塊的全域設定&#41; &#40;AccessToSQL&#41;](../../ssma/access/global-settings-dialogs-accesstosql.md)|使用 [**通用設定**] 對話方塊的 [編輯器] 頁面，即可設定預設對話方塊和警告設定。|  
|[&#40;記錄的全域設定&#41; &#40;AccessToSQL&#41;](../../ssma/access/global-settings-logging-accesstosql.md)|使用 [**通用設定**] 對話方塊的 [記錄] 頁面，即可設定記錄。|  
|[全域設定 &#40;輸出視窗&#41; &#40;AccessToSQL&#41;](../../ssma/access/global-settings-output-window-accesstosql.md)|使用 [**通用設定**] 對話方塊的 [輸出視窗] 頁面，即可設定 [輸出] 窗格和 [錯誤視窗] 窗格的顯示。|  
|[移轉精靈](migration-wizard-accesstosql.md)|使用 [遷移嚮導] 引導您完成遷移程式。|  
|[新增專案 (SSMA)](https://msdn.microsoft.com/ca294f6d-eeb5-42ca-9306-156281a3f0f3)|使用 [**新增專案**] 對話方塊來建立新的 SSMA 專案。|  
|[專案設定 (轉換)](https://msdn.microsoft.com/bcebc635-c638-4ddb-924c-b9ccfef86388)|使用 [**專案設定**] 對話方塊的 [**轉換**] 頁面，即可指定 SSMA 如何轉換索引的備忘資料行、主鍵、外鍵條件約束、時間戳記和沒有索引的資料表。|  
|[&#40;GUI 的專案設定&#41; &#40;AccessToSQL&#41;](../../ssma/access/project-settings-gui-accesstosql.md)|使用 [**專案設定**] 對話方塊的 [GUI] 頁面，即可指定要在 [**資料**] 索引標籤上顯示的資料量。|  
|[專案設定 (移轉)](https://msdn.microsoft.com/4caebc9c-8680-4b99-a8fa-89c43161c95d)|使用 [**專案設定**] 對話方塊的 [遷移] 頁面，即可指定遷移批次大小、資料表鎖定、條件約束檢查、插入觸發程式引發、識別和 null 值處理，以及如何處理超出範圍的日期 [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] 。|  
|[專案設定 (類型對應)](https://msdn.microsoft.com/b87b9683-abed-4677-8c50-18bdba704655)|使用 [**專案設定**] 對話方塊的 [類型對應] 頁面，即可指定 SSMA 專案中所有資料庫和資料庫物件的預設類型對應。|
