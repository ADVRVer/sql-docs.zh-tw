# <a name="s2ss0092-the-conversion-for-provided-date-format-is-not-supported-error"></a>S2SS0092:不支援提供日期格式的轉換(錯誤)

本文介紹為什麼 Sybase 的 SQL Server 移轉助手 (SSMA) 不會將呼叫轉換為具有指定樣式參數的 CONVERT 函式

## <a name="background"></a>背景

每當對`CONVERT`函數調用或從日期/時間數據類型 SSMA 調用時,SSMA 將嘗試適當地翻譯它。 Sybase/SAP ASE 和 SQL Server 支援不同的日期和時間`format-style`格式,因此可能有 (第三個)參數的值在 SQL Server 中沒有直接映射。 SSMA 嘗試使用顯`FORMAT`式 函數調用來類比其中一些樣式,但仍有 SSMA 無法轉換的格式樣式。

## <a name="example"></a>範例

請考慮下面的範例,其中我們嘗試使用樣式設定日期值的格式`30`:

```sql
SELECT
  CONVERT(VARCHAR(20), [StartDate], 30) AS [StartDate]
FROM
  [Employees]
```

當您試著在 SSMA 中轉換此代碼時,它會產生以下錯誤訊息:

> S2SS0092:不支援提供日期格式的轉換

## <a name="possible-remedies"></a>可能的補救措施

要解決此錯誤,可以使用具有所需`FORMAT`格式的函數:

```sql
SELECT
  FORMAT([StartDate], 'dd \d MM \m yyyy \y') AS [StartDate]
FROM
  [Employees]
```
