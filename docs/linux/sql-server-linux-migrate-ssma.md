# <a name="automate-database-migration-to-linux-with-the-sql-server-migration-assistant"></a>自動化與 SQL Server Migration Assistant 的資料庫移轉到 Linux

[!INCLUDE[appliesto-ss-xxxx-xxxx-xxx-md-linuxonly](../includes/appliesto-ss-xxxx-xxxx-xxx-md-linuxonly.md)]

這篇文章介紹[SQL Server Migration Assistant (SSMA)](https://msdn.microsoft.com/library/mt613434.aspx) ，以協助您輕鬆地將資料庫從 Microsoft Access、 DB2、 MySQL、 Oracle 及 Sybase 移轉至 Linux 上的 SQL Server。 SSMA 是 Windows 應用程式，因此使用 SSMA 就可以連線到 Linux 上的遠端 SQL Server 執行個體的 Windows 機器。 

SSMA 支援各種不同的來源資料庫，包括 Oracle、 MySQL、 Sybase、 oracle、db2 和 Linux 上的 SQL server 的 Microsoft Access，並可協助自動化移轉工作，例如：

- 評估您的來源資料庫
- 將來源資料庫結構描述轉換成 Microsoft SQL Server 結構描述
- 移轉結構描述
- 將資料移轉
- 測試移轉

若要開始，下載 SQL Server Migration Assistant (SSMA) 適用於您的來源資料庫，請從下列清單：
- [SSMA for Access](https://aka.ms/ssmaforaccess)
- [SSMA for DB2](https://aka.ms/ssmafordb2)
- [SSMA for MySql](https://aka.ms/ssmaformysql) 
- [SSMA for Oracle](https://aka.ms/ssmafororacle)
- [SSMA for Sybase ASE](https://aka.ms/ssmaforsybase) 

接下來，請遵循[SQL Server Migration Assistant (SSMA)](https://msdn.microsoft.com/library/mt613434.aspx)將來源資料庫移轉到 Linux 上的 SQL Server。
