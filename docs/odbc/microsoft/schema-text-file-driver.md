# <a name="schema-text-file-driver"></a>結構描述 (文字檔驅動程式)
此架構會包含資料來源中每個資料表（文字檔）的相關資訊，包括資料表的格式、要掃描來判斷資料行類型的資料列數目、資料表的第一個資料列是否包含資料行名稱、是否使用 OEM 或 ANSI 字碼頁撰寫原始程式檔，以及每個資料行的名稱、資料類型和寬度。
