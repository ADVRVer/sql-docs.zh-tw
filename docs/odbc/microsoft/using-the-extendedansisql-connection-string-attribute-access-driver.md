# <a name="using-the-extendedansisql-connection-string-attribute-access-driver"></a>使用 ExtendedAnsiSQL 連接字串屬性 (Access 驅動程式)
在 ODBC Desktop 資料庫驅動程式4.0 版中，Jet 4.0 有幾個預設未啟用的新功能。 若要利用這些新功能，應用程式必須設定新的連接屬性、將新的屬性新增至連接字串，或在 DSN 中設定屬性。 此屬性稱為 ExtendedAnsiSQL，僅與 Microsoft ODBC Desktop 資料庫驅動程式4.0 和更新版本相關。  
  
 此章節包含下列主題。  
  
-   [設定 ExtendedAnsiSQL](../../odbc/microsoft/setting-extendedansisql.md)  
  
-   [設定 ExtendedAnsiSQL 來啟用新資料類型](../../odbc/microsoft/enabling-new-data-types-by-setting-extendedansisql.md)  
  
-   [使用 ExtendedAnsiSQL 來建立和卸除檢視和程序](../../odbc/microsoft/creating-and-dropping-views-and-procedures-using-extendedansisql.md)  
  
-   [使用 ExtendedAnsiSQL 啟用的資料 Trunctation 偵測](../../odbc/microsoft/data-truncation-detection-enabled-using-extendedansisql.md)  
  
-   [ExtendedAnsiSQL Set 時，Jet 4.0 會使用 SQL-92 保留字組清單](../../odbc/microsoft/jet-4-0-uses-sql-92-reserved-words-list-when-extendedansisql-set.md)
