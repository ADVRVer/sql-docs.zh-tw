# <a name="odbc-glossary"></a>ODBC 詞彙
## <a name="a"></a>A  
 **存取計劃**  
 資料庫引擎為執行 SQL 語句而生成的計劃。 等效於從第三代語言(如 C)編譯的可執行代碼。  
  
 **彙總函數**  
 從一組值生成單個值的函數,通常與**GROUP BY**和**HAVING**子句一起使用。 聚合函數包括**SUM**AVG、COUNT、MAX、MIN 和**AVG****COUNT****MAX****MIN**SUM 。 也稱為*Ctt 函數*。 *另請參閱*標量函數。  
  
 **ANSI**  
 美國國家標準協會。 ODBC API 基於 ANSI 呼叫層介面。  
  
 **APD**  
 *請參閱*應用程式參數描述符 (APD)。  
  
 **API**  
 應用程式程式設計介面。 應用程式用於請求和執行較低級別的服務的一組例程。 ODBC API 由 ODBC 函數組成。  
  
 **應用程式**  
 呼叫 ODBC API 中的函數的執行程式。  
  
 **應用程式參數描述符號 (APD)**  
 描述在應用程式指定的任何轉換之前 SQL 語句中使用的動態參數的描述符。  
  
 **應用程式行描述符 (ARD)**  
 表示應用程式緩衝區中的列元數據和數據的描述符,描述應用程式指定的任何數據轉換之後的一行數據。  
  
 **阿爾德**  
 *請參閱*應用程式行描述符 (ARD)。  
  
 **自動提交模式**  
 事務提交模式,其中事務在執行后立即提交。  
  
## <a name="b"></a>B  
 **行為變化**  
 某些功能從 ODBC *3.x*行為更改為 ODBC *2.x*行為,反之亦然。 由更改SQL_ATTR_ODBC_VERSION環境屬性引起的。  
  
 **二進位物件 (BLOB)**  
 超過一定位元組數(如255)的任何二進位數據。 通常更長。 此類數據通常以部分的方式發送到數據源並從數據源中檢索。 也稱為*長資料*。  
  
 **綁定**  
 作為動詞,將結果集中的列或 SQL 語句中的參數與應用程式變數相關聯的行為。 作為名詞,協會。  
  
 **繫結位移**  
 添加到所有綁定列或參數數據的數據緩衝區位址和長度/指示器緩衝區位址的值,生成新位址。  
  
 **區塊游標**  
 一次能夠獲取多行數據的游標。  
  
 **緩衝區**  
 用於在應用程式和驅動程式之間傳遞數據的應用程式記憶體。 緩衝區通常成對:*資料緩衝區*與*資料長度緩衝區*。  
  
 **byte**  
 八位或一個八位。 *另請參閱*八進位。  
  
## <a name="c"></a>C  
 **C 資料類型**  
 C 程式中變數的資料類型,在這種情況下,是應用程式。  
  
 **目錄**  
 描述資料庫形狀的資料庫中的系統表集。 將稱為*架構*或*資料字典*。  
  
 **目錄功能**  
 用於從資料庫的目錄中檢索資訊的 ODBC 函數。  
  
 **CLI**  
 *參見*Api。  
  
 **用戶端/伺服器**  
 一種資料庫訪問策略,其中一個或多個用戶端通過伺服器訪問數據。 用戶端通常實現用戶介面,而伺服器控制資料庫訪問。  
  
 **欄**  
 行中單個資訊項的容器。 也稱為*欄位*。  
  
 **提交**  
 使事務中的更改永久化。  
  
 **併發**  
 多個事務同時訪問相同數據的能力。  
  
 **一致性等級**  
 驅動程式或數據源支援的離散功能集。 ODBC 定義 API 一致性級別和 SQL 一致性級別。  
  
 **連線**  
 驅動程序和數據源的特定實例。  
  
 **連線瀏覽**  
 在網路中搜索要連接到的數據源。 連接流覽可能涉及幾個步驟。 例如,使用者可能首先瀏覽網路以獲取伺服器,然後流覽特定伺服器以獲取資料庫。  
  
 **連接句柄**  
 包含有關連接資訊的數據結構的句柄。  
  
 **目前列**  
 游標當前指向的行。 定位操作在當前行上作用。  
  
 **游標**  
 將數據行返回給應用程式的軟體。 可能是以計算機終端上的閃爍游標命名的;正如該游標指示螢幕上的當前位置一樣,結果集中的游標指示結果集中的當前位置。  
  
## <a name="d"></a>D  
 **資料緩衝區**  
 用於傳遞數據的緩衝區。 通常與資料緩衝區關聯的是*資料長度緩衝區*。  
  
 **資料字典**  
 *請參閱*目錄。  
  
 **資料長度緩衝區**  
 用於傳遞相應*數據緩衝區*中值長度的緩衝區。 數據長度緩衝區還用於儲存指示器,例如數據值是否為 null 終止。  
  
 **資料來源**  
 使用者要造訪的數據及其關聯的作業系統、DBMS和網路平臺(如果有)。  
  
 **資料類型**  
 數據片段的類型。 ODBC 定義 C 和 SQL 資料類型。 *另請參閱*類型指示器。  
  
 **執行時的資料欄**  
 呼叫**SQLSetPos**後為其發送數據的列。 因此命名是因為數據在執行時發送,而不是放置在行集緩衝區中。 長數據通常在執行時以零件發送。  
  
 **執行時的資料參數**  
 呼叫**SQLExecute**或**SQLExecDirect**後發送數據的參數。 之所以如此命名,是因為數據是在執行 SQL 語句而不是放置在參數緩衝區中時發送的。 長數據通常在執行時以零件發送。  
  
 **資料庫**  
 DBMS 中的離散數據集合。 也是 DBMS。  
  
 **資料庫引擎**  
 DBMS 中用於解析和執行 SQL 語句並造訪實體資料的軟體。  
  
 **Dbms**  
 資料庫管理系統。 實體資料庫與使用者之間的軟體層。 DBMS 會管理資料庫的所有存取。  
  
 **基於 DBMS 驅動程式**  
 通過獨立資料庫引擎訪問物理數據的驅動程式。  
  
 **DDL**  
 數據定義語言。 SQL 中定義而不是操作數據的語句。 例如,**建立表**格、**建立索引**、**請建立**與**撤銷**。  
  
 **分隔識別碼**  
 包含在標識符報價字元中的標識碼,以便它可以包含特殊字元或匹配關鍵字(也稱為報價識別碼)。  
  
 **描述符號**  
 保存有關列數據或動態參數的資訊的數據結構。 描述符的物理表示形式未定義;因此,描述符的物理表示形式未定義。應用程式只能通過使用描述符句柄調用 ODBC 函數來操作其欄位,才能直接訪問描述符。  
  
 **桌面資料庫**  
 設計為在個人電腦上運行的 DBMS。 通常,這些 DBMS 不提供獨立的資料庫引擎,必須通過基於檔的驅動程式進行訪問。 這些驅動程式中的引擎通常減少了對 SQL 和事務的支援。 例如,dBASE、悖論、Btrieve 或微軟®福克斯專業®。  
  
 **診斷**  
 包含有關使用特定句柄的最後一個函數的診斷資訊的記錄。 診斷記錄與環境、連接、語句和描述符句柄相關聯。  
  
 **DML**  
 數據操作語言。 SQL 中的那些語句操作數據而不是定義數據。 例如,**插入**,**DELETE****請移除**與**選擇**。  
  
 **司機**  
 一個例程庫,用於公開 ODBC API 中的函數。 驅動程式特定於單個 DBMS。  
  
 **驅動程式管理員**  
 管理對應用程式驅動程式的訪問的例程庫。 驅動程式管理員載入和卸載(或連接到和斷開)驅動程式,並將對 ODBC 函數的調用傳遞給正確的驅動程式。  
  
 **驅動程式設定 DLL**  
 包含特定於驅動程式的安裝和配置功能的 DLL。  
  
 **動態資料指標**  
 能夠檢測在結果集中更新、刪除或插入的行的可滾動游標。  
  
 **動態 SQL**  
 在其中創建和編譯 SQL 語句的嵌入式 SQL 類型。 *另請參閱*靜態 SQL。  
  
## <a name="e"></a>E  
 **內嵌式 SQL**  
 直接包含在用其他語言(如 COBOL 或 C. ODBC)編寫的程式中的 SQL 語句不使用嵌入式 SQL。 *另請參閱*靜態 SQL*和*動態 SQL。  
  
 **環境**  
 訪問數據的全域上下文;與環境關聯的是任何全域資訊,例如該環境中所有連接的清單。  
  
 **環境控點**  
 包含有關環境資訊的數據結構的句柄。  
  
 **轉義條款**  
 SQL 語句中子句。  
  
 **執行**  
 運行 SQL 語句。  
  
## <a name="f"></a>F  
 **脂肪游標**  
 *請參閱*塊游標。  
  
 **抓取**  
 從結果集中檢索一個或多個行。  
  
 **領域**  
 *請參閱*列。  
  
 **基於檔案的驅動程式**  
 直接存取物理數據的驅動程式。 在這種情況下,驅動程式包含資料庫引擎,並同時充當驅動程式和數據源。  
  
 **檔案資料來源**  
 儲存連線資訊在 .dsn 檔案中的數據來源。  
  
 **外部索引鍵 (foreign key)**  
 表中匹配另一個表中的主鍵的列。  
  
 **只轉寄游標**  
 只能向前移動的結果集,並且通常一次只能獲取一行的游標。 大多數關係資料庫僅支援僅轉發游標。  
  
## <a name="h"></a>H  
 **處理**  
 唯一標識檔或數據結構等內容的值。 句柄僅對創建和使用它們的軟體有意義,但由其他軟體傳遞以識別內容。 ODBC 定義環境、連接、語句和描述符的句柄。  
  
## <a name="i"></a>I  
 **設定參數描述子 (IPD)**  
 描述在應用程式指定的任何轉換後 SQL 語句中使用的動態參數的描述符。  
  
 **實作行描述符號 (IRD)**  
 描述應用程式指定的任何轉換之前的一行資料的描述符。  
  
 **安裝程式 DLL**  
 安裝 ODBC 元件並配置資料來源的 DLL。  
  
 **誠信增強設施**  
 旨在維護資料庫完整性的 SQL 子集。  
  
 **介面一致性層級**  
 驅動程式支援的 ODBC 3.7 介面的級別;可以是核心、級別 1 或級別 2。  
  
 **互通性**  
 一個應用程式在存取不同 DBMS 中的數據時使用相同的代碼的能力。  
  
 **IPD**  
 *參見*實現參數描述符 (IPD)。  
  
 **稅務局**  
 *參見*實現行描述符 (IRD)。  
  
 **ISO/IEC**  
 國際標準組織/國際電工委員會。 ODBC API 基於 ISO/IEC 呼叫層介面。  
  
## <a name="j"></a>J  
 **加入**  
 關係資料庫中的一個操作,通過匹配指定列中的值來連結兩個或多個表中的行。  
  
## <a name="k"></a>K  
 **關鍵**  
 其值標識行的列。 *另請參閱*外鍵*和*主鍵。  
  
 **鍵集**  
 混合或按鍵集驅動的游標用於重新提取行的一組鍵。  
  
 **索引鍵集驅動資料指標**  
 使用鍵集檢測更新和刪除的行的可滾動游標。  
  
## <a name="l"></a>L  
 **文字**  
 SQL 語句中實際數據值的字元表示形式。  
  
 **鎖定**  
 DBMS 限制對多用戶環境中的行的訪問的過程。 DBMS 通常設置一行或物理頁上的位,其中包含指示該行或頁面已鎖定的行。  
  
 **長資料**  
 超過特定長度的任何二進位或字元資料,例如 255 位元組或字元。 通常更長。 此類數據通常以部分的方式發送到數據源並從數據源中檢索。 也稱為*BLOB*或*CLOB。*  
  
## <a name="m"></a>M  
 **機器資料來源**  
 存儲連接資訊的數據源(例如註冊表)。  
  
 **手動提交模式**  
 事務提交模式,其中事務必須通過調用**SQLTransact**顯式提交。  
  
 **元**  
 描述 SQL 語句中的參數或結果集中的欄資料。 例如,參數的數據類型、位元組長度和精度。  
  
 **多層驅動程式**  
 *參見*基於 DBMS 的驅動程式。  
  
## <a name="n"></a>N  
 **空值**  
 沒有顯式分配的值。 特別是,NULL 值不同於零或空白。  
  
## <a name="o"></a>O  
 **八點**  
 八位或一個字節。 *另請參閱*位元組。  
  
 **八進位長度**  
 緩衝區的長度(以八位位元組為單位)或其包含的數據。  
  
 **ODBC**  
 打開資料庫連接。 API 的規範,用於定義應用程式可以存取資料來源中資料的標準程式程集。  
  
 **ODBC 管理員**  
 呼叫安裝程式 DLL 以設定資料來源的執行程式。  
  
 開啟群組  
 發佈標準的公司。 特別是,它發佈 SQL 訪問組 (SAG) 標準。  
  
 **樂觀併發**  
 增加行未鎖定的併發的策略。 相反,在更新或刪除它們之前,游標會檢查它們自上次讀取以來是否已更改。 如果是這樣,更新或刪除將失敗。 *另請參閱*悲觀併發。  
  
 **outer join**  
 返回匹配行和非匹配行的聯接。 不匹配行中不匹配表中所有列的值都設置為 NULL。  
  
 **擁有者**  
 表的所有者。  
  
## <a name="p"></a>P  
 **參數**  
 SQL 語句中的變數,標有參數標記或問號 (?)。 參數綁定到應用程式變數,並在執行語句時檢索其值。  
  
 **參數描述符號**  
 描述 SQL 語句中使用的執行時參數的描述符,無論是在應用程式指定的任何轉換(應用程式參數描述符或 APD)之前,還是在應用程式指定的任何轉換(實現參數描述符或 IPD)之後。  
  
 **參數操作陣列**  
 包含應用程式可以設置為指示應在**SQLExecDirect**或**SQLExecute**操作中忽略相應參數的值的陣列。  
  
 **參數狀態陣列**  
 在調用**SQLExecDirect**或**SQLExecute**後包含參數狀態的陣列。  
  
 **封閉式並行存取**  
 實現序列化的策略,其中行被鎖定,以便其他事務無法更改它們。 *另請參閱*樂觀併發*性和*序列化性。  
  
 **定位動作**  
 對當前行運行的任何操作。 例如,定位更新和移除語句 **、SQLGetData**和**SQLSetPos**。  
  
 **定位更新敘述**  
 用於更新目前行中的值的 SQL 語句。  
  
 **請選擇移除敘述**  
 用於刪除目前的行的 SQL 語句。  
  
 **準備**  
 編譯 SQL 語句。 通過準備 SQL 語句創建訪問計畫。  
  
 **主鍵**  
 唯一標識表中行的列。  
  
 **程式**  
 一組一個或多個預編譯的 SQL 語句,這些語句作為命名物件存儲在資料庫中。  
  
 **過程欄**  
 過程呼叫中的參數、過程返回的值或過程創建的結果集中的列。  
  
## <a name="q"></a>Q  
 **限定詞**  
 包含一個或多個表的資料庫。  
  
 **查詢**  
 SQL 語句。 有時用於表示**SELECT**語句。  
  
 **引號識別碼**  
 包含在識別碼報價字元中的識別碼,以便它可以包含特殊字元或匹配關鍵字(在 SQL-92 中也稱為分隔識別符)。  
  
## <a name="r"></a>R  
 **拉迪克斯**  
 數位系統的基礎。 通常為 2 或 10。  
  
 **記錄**  
 *請參閱*行。  
  
 **結果集**  
 通過執行**SELECT**語句創建的行集。  
  
 **返回代碼**  
 ODBC 函數返回的值。  
  
 **回復**  
 將事務更改的值返回到其原始狀態。  
  
 **列**  
 描述特定實體的一組相關列。 也稱為*記錄*。  
  
 **行描述子**  
 描述結果集列的描述符,在應用程式指定的任何轉換(實現行描述符或 IRD)之前,或在應用程式指定的任何轉換(應用程式行描述符或 ARD)之後。  
  
 **列操作陣列**  
 包含應用程式可以設置為指示應在**SQLSetPos**操作中忽略相應行的值的陣列。  
  
 **行狀態陣列**  
 在調用**SQLFetch、SQLFetchScroll**或**SQLSetPos**後包含行狀態**SQLFetchScroll**的陣列。  
  
 **排集**  
 塊游標在單個提取中返回的行集。  
  
 **排集緩衝區**  
 綁定到結果集的列的緩衝區,並在其中返回整個行集的數據。  
  
## <a name="s"></a>S  
 **凹陷**  
 *參見*SQL 訪問組 (SAG)。  
  
 **標量函數**  
 從單個值生成單個值的函數。 例如,更改字元數據的項例的函數。  
  
 **模式**  
 *請參閱*目錄。  
  
 **可捲軸游標**  
 可在結果集中向前或向後移動的游標。  
  
 **序列化**  
 是否同時執行兩個事務會產生與這些事務的串列(或順序)執行相同的結果。 需要序列化事務來維護資料庫完整性。  
  
 **伺服器資料庫**  
 設計為在用戶端/伺服器環境中運行的 DBMS。 這些 DBMS 提供了一個獨立的資料庫引擎,為 SQL 和事務提供豐富的支援。 它們通過基於 DBMS 的驅動程式進行訪問。 例如,Oracle、Informix、DB/2 或 Microsoft ® SQL 伺服器。  
  
 **設定功能**  
 *請參閱*聚合函數。  
  
 **設定 DLL**  
 *請參閱*驅動程式設定 DLL*和*轉換器設定 DLL。  
  
 **單層驅動程式**  
 *請參閱*根據檔的驅動程式。  
  
 **SQL**  
 結構化查詢語言。 關係資料庫用於查詢、更新和管理數據的語言。  
  
 **SQL 存取群組 (SAG)**  
 與 SQL DBMS 相關的公司行業聯盟。 開放組的呼叫級介面基於最初由 SQL 訪問組完成的工作。  
  
 **SQL 符合性層級**  
 驅動程式支援的 SQL-92 語法級別;可以是入口、FIPS過渡、中間或完整。  
  
 **SQL 資料類型**  
 列或參數的數據類型,因為它存儲在數據源中。  
  
 **SQLSTATE**  
 指示特定錯誤的五個字元值。  
  
 **SQL 語句**  
 SQL 中以關鍵字開頭並完全描述要執行的操作的完整短語。 例如,選擇 = 從訂單。 SQL 語句不應與語句混淆。  
  
 **state**  
 項的已定義條件。 例如,連接有七種狀態,包括未分配、分配、連接和需要的數據。 僅當項處於特定狀態時,才能執行某些操作。 例如,僅當連接處於分配狀態時,才能釋放連接,例如,當連接處於連接狀態時,才能釋放。  
  
 **狀態轉換**  
 項從一種狀態到另一種狀態的移動。 ODBC 為環境、連接和語句定義嚴格的狀態轉換。  
  
 **宣告**  
 與 SQL 語句相關的所有資訊的容器。 語句不應與 SQL 語句混淆。  
  
 **語句句柄**  
 包含語句信息的數據結構的句柄。  
  
 **靜態資料指標**  
 無法偵測結果集中的更新、刪除或插入的可滾動游標。 通常通過複製結果集實現。  
  
 **靜態 SQL**  
 一種嵌入式 SQL,其中 SQL 語句在編譯程式的其餘部分時進行硬編碼和編譯。 *另請參閱*動態 SQL。  
  
 **預存程序**  
 *請參閱*過程。  
  
## <a name="t"></a>T  
 **表**  
 行的集合。  
  
 **特恩金**  
 當 16 位元應用程式與 32 位元 ODBC 驅動程式一起使用時,將 16 位元位址轉換為 32 位元位址,反之亦然。  
  
 **交易**  
 不可部分完成的工作單位。 交易中必須整體完成的工作；如果交易的任何部分失敗，整個交易都會失敗。  
  
 **交易隔離**  
 將一個事務與所有其他事務的影響隔離開來的行為。  
  
 **交易隔離等級**  
 衡量事務隔離程度的度量。 有五個事務隔離級別:讀取未提交、已讀、可重複讀取、可序列化和版本控制。  
  
 **譯者 DLL**  
 用於將數據從一個字元集轉換為另一個字元集的 DLL。  
  
 **轉換器設定 DLL**  
 包含特定於轉換器的安裝和配置功能的 DLL。  
  
 **兩階段提交**  
 分兩個階段提交分散式事務的過程。 在第一階段,事務處理器檢查事務的所有部分是否可以提交。 在第二階段,事務的所有部分都提交。 如果事務的任何部分在第一階段指示無法提交,則第二階段不會發生。 ODBC 不支援兩階段提交。  
  
 **型態指示器**  
 傳遞給或從 ODBC 函數傳回的整數值,用於指示應用程式變數、參數或列的數據類型。 ODBC 為 C 和 SQL 資料類型定義類型指示器。  
  
## <a name="v"></a>V  
 **檢視**  
 查看一個或多個表中的數據的另一種方法。 視圖通常創建為一個或多個表中的列的子集。 在 ODBC 中,視圖通常等效於表。
