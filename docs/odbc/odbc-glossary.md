# <a name="odbc-glossary"></a>ODBC 詞彙
## <a name="a"></a>A  
 **存取方案**  
 資料庫引擎為了執行 SQL 語句所產生的計畫。 相當於從第三代語言（例如 C）所編譯的可執行程式碼。  
  
 **aggregate 函數**  
 從值群組產生單一值的函式，通常與 **GROUP BY** 和 **HAVING** 子句一起使用。 彙總函式包括 **AVG**、 **COUNT**、 **MAX**、 **MIN**和 **SUM**。 也稱為「 *設定函數」（set 函數*）。 *另請參閱* 純量函數。  
  
 **ANSI**  
 美國國家標準局。 ODBC API 是以 ANSI 呼叫層級介面為基礎。  
  
 **APD**  
 *請參閱* (APD) 的應用程式參數描述項。  
  
 **API**  
 應用程式設計介面。 一組常式，應用程式用來要求和執行較低層級的服務。 ODBC API 是由 ODBC 函數所組成。  
  
 **應用程式**  
 可執行程式，可在 ODBC API 中呼叫函式。  
  
 **應用程式參數描述項 (APD) **  
 描述在應用程式所指定的任何轉換之前，用於 SQL 語句中的動態參數的描述元。  
  
 **應用程式資料列描述元 (ARD) **  
 描述資料行中繼資料和應用程式緩衝區中資料的描述項，描述遵循應用程式所指定的任何資料轉換的資料列。  
  
 **ARD**  
 *請參閱* 應用程式資料列描述項 (ARD) 。  
  
 **自動認可模式**  
 交易認可模式，在此模式下，交易會在執行之後立即認可。  
  
## <a name="b"></a>B  
 **行為變更**  
 從 ODBC 3.x*行為到 odbc* 2.x*行為的*特定功能變更，反之亦然。 變更 SQL_ATTR_ODBC_VERSION 環境屬性所造成。  
  
 **二進位大型物件 (BLOB) **  
 特定位元組數目的任何二進位資料，例如255。 通常會更長。 這類資料通常會傳送至元件中的資料來源並從中取出。 也稱為 *long 資料*。  
  
 **綁定**  
 作為動詞，將結果集內的資料行或 SQL 語句中的參數與應用程式變數相關聯的動作。 作為名詞，關聯。  
  
 **系結位移**  
 針對所有系結的資料行或參數資料新增至資料緩衝區位址和長度/指標緩衝區位址的值，會產生新的位址。  
  
 **區塊游標**  
 一次能夠提取多個資料列的資料指標。  
  
 **緩衝區**  
 用來在應用程式與驅動程式之間傳遞資料的應用程式記憶體片段。 緩衝區通常會成對： *資料緩衝區* 和 *資料長度緩衝區*。  
  
 **byte**  
 八位或一個八位。 *另請參閱* 八位。  
  
## <a name="c"></a>C  
 **C 資料類型**  
 C 程式中變數的資料類型，在此案例中為應用程式。  
  
 **catalog**  
 資料庫中的一組系統資料表，描述資料庫的圖形。 也稱為 *架構* 或 *資料字典*。  
  
 **catalog 函數**  
 用來從資料庫目錄中取得資訊的 ODBC 函數。  
  
 **CLI**  
 *請參閱* Api。  
  
 **用戶端/伺服器**  
 一或多個用戶端透過伺服器存取資料的資料庫存取策略。 用戶端通常會在伺服器控制資料庫存取時，執行使用者介面。  
  
 **column**  
 資料列中單一資訊專案的容器。 也稱為「 *欄位*」。  
  
 **提交**  
 使交易中的變更永久有效。  
  
 **併發**  
 有多個交易同時存取相同資料的能力。  
  
 **一致性層級**  
 驅動程式或資料來源所支援的一組獨立功能。 ODBC 會定義 API 一致性層級和 SQL 一致性層級。  
  
 **connection**  
 驅動程式和資料來源的特定實例。  
  
 **連接流覽**  
 搜尋網路以取得要連接的資料來源。 連接流覽可能牽涉到幾個步驟。 例如，使用者可能會先流覽網路中的伺服器，然後流覽特定伺服器的資料庫。  
  
 **連接控制碼**  
 資料結構的控制碼，其中包含連接的相關資訊。  
  
 **目前的資料列**  
 資料指標目前指向的資料列。 定位作業作用於目前的資料列。  
  
 **cursor**  
 將資料列傳回給應用程式的一段軟體。 可能是在電腦終端機上閃爍的游標之後命名;就像該資料指標表示螢幕上目前的位置一樣，結果集上的資料指標也會指出結果集內的目前位置。  
  
## <a name="d"></a>D  
 **資料緩衝區**  
 用來傳遞資料的緩衝區。 通常與資料緩衝區相關聯的是 *資料長度緩衝區*。  
  
 **資料字典**  
 *請參閱* 目錄。  
  
 **資料長度緩衝區**  
 用來在對應的 *資料緩衝區*中傳遞值長度的緩衝區。 資料長度緩衝區也用來儲存指標，例如資料值是否以 null 終止。  
  
 **資料來源**  
 使用者想要存取的資料，以及其相關聯的作業系統、DBMS 和網路平臺 (（如果有任何) ）。  
  
 **資料類型**  
 資料片段的類型。 ODBC 會定義 C 和 SQL 資料類型。 *另請參閱* 類型指標。  
  
 **資料執行中資料行**  
 呼叫 **SQLSetPos** 之後傳送資料的資料行。 因為資料會在執行時間傳送，而不是放在資料列集緩衝區中，所以命名為。 長資料通常會在執行時間于元件中傳送。  
  
 **資料執行中參數**  
 呼叫 **SQLExecute** 或 **SQLExecDirect** 之後傳送資料的參數。 因為會在執行 SQL 語句時傳送資料，而不是放在參數緩衝區中，所以命名為。 長資料通常會在執行時間于元件中傳送。  
  
 **database**  
 DBMS 中的離散資料集合。 也是 DBMS。  
  
 **資料庫引擎**  
 DBMS 中的軟體，可剖析和執行 SQL 語句，並存取實體資料。  
  
 **Dbms**  
 資料庫管理系統。 實體資料庫與使用者之間的軟體層。 DBMS 會管理資料庫的所有存取。  
  
 **以 DBMS 為基礎的驅動程式**  
 透過獨立資料庫引擎存取實體資料的驅動程式。  
  
 **DDL**  
 資料定義語言。 SQL 中的語句，這些語句會定義，而不是運算元據。 例如， **CREATE TABLE**、 **CREATE INDEX**、 **GRANT**和 **REVOKE**。  
  
 **分隔識別碼**  
 以識別碼引號字元括住的識別碼，可包含特殊字元或符合關鍵字 (也稱為引號識別碼) 。  
  
 **描述 符**  
 保存資料行資料或動態參數相關資訊的資料結構。 未定義描述項的實體標記法;應用程式只能藉由使用描述項控制碼來呼叫 ODBC 函數，藉以直接存取描述項。  
  
 **桌面資料庫**  
 專為在個人電腦上執行而設計的 DBMS。 一般而言，這些 Dbms 不提供獨立的資料庫引擎，必須透過以檔案為基礎的驅動程式存取。 這些驅動程式中的引擎通常會降低對 SQL 和交易的支援。 例如，dBASE、Paradox、Btrieve 或 Microsoft® FoxPro®。  
  
 **診斷**  
 記錄，其中包含使用特定控制碼之最後一個函式的相關診斷資訊。 診斷記錄會與環境、連接、語句和描述項控制碼相關聯。  
  
 **DML**  
 資料操作語言。 SQL 中可操作的語句，而不是定義資料。 例如， **插入**、 **更新**、 **刪除**及 **選取**。  
  
 **司機**  
 公開 ODBC API 中之函式的常式程式庫。 驅動程式專用於單一 DBMS。  
  
 **驅動程式管理員**  
 管理應用程式之驅動程式存取權的常式程式庫。 驅動程式管理員會載入和卸載 (，或連接到) 驅動程式並中斷連接，並將 ODBC 函數的呼叫傳遞給正確的驅動程式。  
  
 **驅動程式安裝程式 DLL**  
 包含驅動程式特定安裝和設定功能的 DLL。  
  
 **動態資料指標**  
 可以在結果集中偵測更新、刪除或插入資料列的可滾動資料指標。  
  
 **動態 SQL**  
 內嵌 SQL 的類型，會在執行時間建立和編譯 SQL 語句。 另*請參閱*靜態 SQL。  
  
## <a name="e"></a>E  
 **內嵌 SQL**  
 直接包含在以另一種語言（例如 COBOL 或 c.）撰寫之程式中的 SQL 語句，不會使用內嵌的 SQL。 另*請參閱*靜態 sql*和*動態 sql。  
  
 **environment (環境)**  
 要在其中存取資料的全域內容;與環境相關聯的任何資訊都是全域性質的資訊，例如該環境中的所有連線清單。  
  
 **環境控制碼**  
 資料結構的控制碼，其中包含環境的相關資訊。  
  
 **escape 子句**  
 SQL 語句中的子句。  
  
 **執行**  
 以執行 SQL 語句。  
  
## <a name="f"></a>F  
 **fat 資料指標**  
 *請參閱* 區塊資料指標。  
  
 **獲取**  
 從結果集取出一或多個資料列。  
  
 **領域**  
 *請參閱* 資料行。  
  
 **以檔案為基礎的驅動程式**  
 直接存取實體資料的驅動程式。 在此情況下，驅動程式會包含資料庫引擎，並同時作為驅動程式和資料來源。  
  
 **檔案資料來源**  
 連接資訊儲存在 dsn 檔案中的資料來源。  
  
 **外鍵**  
 資料表中的一或多個資料行，與另一個資料表中的主鍵相符。  
  
 **順向資料指標**  
 只能向前移動結果集的資料指標，且通常只會一次提取一個資料列。 大部分的關係資料庫只支援順向資料指標。  
  
## <a name="h"></a>H  
 **處理**  
 可唯一識別某個事物（例如檔案或資料結構）的值。 控制碼只對建立和使用的軟體有意義，但會由其他軟體傳遞，以找出事物。 ODBC 定義了環境、連接、語句和描述項的控制碼。  
  
## <a name="i"></a>I  
 ** (IPD) 的實參數描述項 **  
 描述在應用程式所指定的任何轉換之後，在 SQL 語句中使用之動態參數的描述元。  
  
 **IRD) 的實資料列描述元 (**  
 描述應用程式所指定之任何轉換之前的資料列的描述元。  
  
 **安裝程式 DLL**  
 安裝 ODBC 元件和設定資料來源的 DLL。  
  
 **完整性增強功能**  
 SQL 的子集，其設計目的是維護資料庫的完整性。  
  
 **介面一致性層級**  
 驅動程式支援的 ODBC 3.7 介面層級;可以是核心、層級1或層級2。  
  
 **互 操作 性**  
 在不同 Dbms 中存取資料時，一個應用程式使用相同程式碼的能力。  
  
 **IPD**  
 *請參閱* (IPD) 的實參數描述項。  
  
 **稅務局**  
 *請參閱* IRD)  (的實資料列描述項。  
  
 **ISO/IEC**  
 國際標準組織/國際電子電機委員會。 ODBC API 是以 ISO/IEC 呼叫層級介面為基礎。  
  
## <a name="j"></a>J  
 **join**  
 關係資料庫中的作業，藉由比對指定資料行中的值，連結兩個或多個資料表中的資料列。  
  
## <a name="k"></a>K  
 **key**  
 值識別資料列的一或多個資料行。 *另請參閱* 外鍵 *和* 主要金鑰。  
  
 **該行**  
 混合或索引鍵集驅動資料指標用來重新擷取資料列的一組索引鍵。  
  
 **索引鍵集驅動資料指標**  
 可滾動的資料指標，可使用索引鍵集來偵測已更新和已刪除的資料列。  
  
## <a name="l"></a>L  
 **literal**  
 SQL 語句中實際資料值的字元標記法。  
  
 **鎖定**  
 DBMS 在多使用者環境中限制存取資料列的進程。 DBMS 通常會在資料列上設定一個位，或在包含表示資料列或頁面鎖定之資料列的實體頁面上設定一個位。  
  
 **長資料**  
 特定長度的任何二進位或字元資料，例如255個位元組或字元。 通常會更長。 這類資料通常會傳送至元件中的資料來源並從中取出。 也稱為 *BLOB*s 或 *CLOB*s。  
  
## <a name="m"></a>M  
 **機器資料來源**  
 將連接資訊儲存在系統 (例如登錄) 的資料來源。  
  
 **手動認可模式**  
 交易認可模式，在此模式中，交易必須藉由呼叫 **SQLTransact**來明確地認可。  
  
 **元**  
 描述 SQL 語句中的參數或結果集內之資料行的資料。 例如，資料類型、位元組長度和參數的有效位數。  
  
 **多層式驅動程式**  
 *請參閱* 以 DBMS 為基礎的驅動程式。  
  
## <a name="n"></a>N  
 **Null 值**  
 沒有明確指派的值。 尤其是，Null 值不同于零或空白。  
  
## <a name="o"></a>O  
 **八進位**  
 八位或一個位元組。 *另請參閱* byte。  
  
 **八位長度**  
 緩衝區的長度，或其包含的資料。  
  
 **ODBC**  
 開啟資料庫連接。 API 的規格，此規格會定義一組標準常式，讓應用程式可以存取資料來源中的資料。  
  
 **ODBC 管理員**  
 呼叫安裝程式 DLL 以設定資料來源的可執行程式。  
  
 開啟群組  
 發佈標準的公司。 尤其是，它會發佈 SQL 存取群組 (SAG) 標準。  
  
 **開放式平行存取**  
 此策略可增加資料列未鎖定的平行存取。 相反地，在更新或刪除之前，資料指標會先檢查它們是否已在上次讀取之後變更。 如果是，更新或刪除會失敗。 *另請參閱* 封閉式平行存取。  
  
 **outer join**  
 同時傳回相符和不相符資料列的聯結。 不相符資料列中不相符資料表的所有資料行值會設定為 Null。  
  
 **擁有者**  
 資料表的擁有者。  
  
## <a name="p"></a>P  
 **參數**  
 SQL 語句中的變數標示了參數標記或問號 (？ ) 。 參數會系結至應用程式變數，並在執行語句時抓取它們的值。  
  
 **參數描述項**  
 描述 SQL 語句中所使用之執行時間參數的描述項，這些參數是在應用程式所指定的任何轉換 (應用程式參數) 描述元之前，或在應用程式指定的任何轉換之後， (執行參數描述元或 IPD) 。  
  
 **參數運算元組**  
 陣列，其中包含應用程式可以設定的值，指出應該在 **SQLExecDirect** 或 **SQLExecute** 作業中忽略對應的參數。  
  
 **參數狀態陣列**  
 陣列，其中包含呼叫 **SQLExecDirect** 或 **SQLExecute**之後的參數狀態。  
  
 **封閉式並行存取**  
 用來執行可序列化能力的策略，其中資料列會被鎖定，讓其他交易無法變更它們。 *另請參閱* 開放式平行存取 *和* 可序列化能力。  
  
 **定位作業**  
 在目前資料列上運作的任何作業。 例如，定位 update 和 delete 語句、 **SQLGetData**和 **SQLSetPos**。  
  
 **定位的 update 語句**  
 用來更新目前資料列中之值的 SQL 語句。  
  
 **定位 delete 語句**  
 用來刪除目前資料列的 SQL 語句。  
  
 **準備**  
 編譯 SQL 語句。 存取計畫是藉由準備 SQL 語句所建立。  
  
 **主要金鑰**  
 在資料表中唯一識別資料列的資料行。  
  
 **程式**  
 一或多個先行編譯 SQL 語句的群組，這些語句會儲存為資料庫中的命名物件。  
  
 **程式資料行**  
 程序呼叫中的引數、程式所傳回的值，或程式所建立之結果集內的資料行。  
  
## <a name="q"></a>Q  
 **限定 符**  
 包含一或多個資料表的資料庫。  
  
 **查詢**  
 SQL 語句。 有時用來表示 **SELECT** 語句。  
  
 **引號識別碼**  
 以識別碼引號字元括住的識別碼，可包含特殊字元或符合關鍵字 (在 SQL-92 中也稱為分隔識別碼) 。  
  
## <a name="r"></a>R  
 **基數**  
 數字系統的基底。 通常是2或10。  
  
 **記錄**  
 *請參閱* 資料列。  
  
 **結果集**  
 藉由執行 **SELECT** 語句所建立的資料列集。  
  
 **傳回碼**  
 ODBC 函數所傳回的值。  
  
 **回復**  
 將交易所變更的值傳回至其原始狀態。  
  
 **行**  
 一組描述特定實體的相關資料行。 也稱為 *記錄*。  
  
 **資料列描述項**  
 描述結果集之資料行的描述項，可以在應用程式所指定的任何轉換 (實資料列) 描述項之前，或在應用程式所指定的任何轉換 (應用程式資料列描述元或 ARD) 之後進行。  
  
 **資料列運算陣列**  
 陣列，其中包含應用程式可以設定的值，指出應該在 **SQLSetPos** 作業中忽略對應的資料列。  
  
 **資料列狀態陣列**  
 陣列，其中包含呼叫 **SQLFetch**、 **SQLFetchScroll**或 **SQLSetPos**之後的資料列狀態。  
  
 **rowset**  
 區塊資料指標在單一提取中傳回的資料列集。  
  
 **資料列集緩衝區**  
 系結至結果集資料行的緩衝區，以及會傳回整個資料列集之資料的緩衝區。  
  
## <a name="s"></a>S  
 **凹陷**  
 *請參閱* SQL 存取群組 (SAG) 。  
  
 **純量函數**  
 從單一值產生單一值的函數。 例如，變更字元資料大小寫的函式。  
  
 **模式**  
 *請參閱* 目錄。  
  
 **可滾動資料指標**  
 可以向前或向後移動結果集的資料指標。  
  
 **可序列化能力**  
 兩個執行的交易是否會同時產生與序列 (相同的結果，或執行這些交易的順序) 。 需要可序列化交易，才能維持資料庫完整性。  
  
 **伺服器資料庫**  
 設計成在用戶端/伺服器環境中執行的 DBMS。 這些 Dbms 提供可提供 SQL 和交易豐富支援的獨立資料庫引擎。 它們是透過 DBMS 驅動程式來存取。 例如，Oracle、Informix、DB/2 或 Microsoft® SQL Server。  
  
 **set 函數**  
 *請參閱* 彙總函式。  
  
 **安裝程式 DLL**  
 *請參閱* 驅動程式安裝 dll *和* 轉譯程式安裝程式 dll。  
  
 **單層式驅動程式**  
 *請參閱* 以檔案為基礎的驅動程式。  
  
 **SQL**  
 結構化查詢語言 (SQL) 。 關係資料庫用來查詢、更新和管理資料的語言。  
  
 **SQL 存取群組 (SAG) **  
 與 SQL Dbms 相關之公司的產業聯盟。 開啟群組的呼叫層級介面是以 SQL 存取群組最初執行的工作為基礎。  
  
 **SQL 一致性層級**  
 驅動程式支援的 SQL-92 文法層級;可以是專案、FIPS 轉換、中繼或完整。  
  
 **SQL 資料類型**  
 資料行或參數儲存在資料來源中的資料類型。  
  
 **SQLSTATE**  
 表示特定錯誤的五個字元的值。  
  
 **SQL 語句**  
 SQL 中以關鍵字開頭的完整片語，並完整描述要採取的動作。 例如，SELECT * FROM Orders。 SQL 語句不應該與語句混淆。  
  
 **state**  
 專案的妥善定義條件。 例如，連線有七種狀態，包括未配置、已配置、已連接和需要的資料。 只有在專案處於特定狀態時，才能執行特定作業。 例如，只有當連接處於已配置的狀態時，才可以釋出，例如，當它處於已線上狀態時。  
  
 **狀態轉換**  
 將專案從某個狀態移至另一個狀態。 ODBC 會針對環境、連接和語句定義嚴謹的狀態轉換。  
  
 **陳述式**  
 所有與 SQL 語句相關之資訊的容器。 語句不應該與 SQL 語句混淆。  
  
 **語句控制碼**  
 包含語句相關資訊之資料結構的控制碼。  
  
 **靜態資料指標**  
 無法偵測結果集中的更新、刪除或插入的可滾動資料指標。 通常是藉由建立結果集的複本來執行。  
  
 **靜態 SQL**  
 內嵌 SQL 的型別，在編譯器的其餘部分時，會以硬式編碼和編譯 SQL 語句。 另*請參閱*動態 SQL。  
  
 **預存程式**  
 *請參閱* 程式。  
  
## <a name="t"></a>T  
 **table**  
 資料列的集合。  
  
 **Thunking**  
 當16位應用程式與32位 ODBC 驅動程式搭配使用時，將16位位址轉換成32位位址，反之亦然。  
  
 **交易**  
 不可部分完成的工作單位。 交易中必須整體完成的工作；如果交易的任何部分失敗，整個交易都會失敗。  
  
 **交易隔離**  
 將一個交易與所有其他交易的效果隔離的動作。  
  
 **交易隔離等級**  
 交易隔離程度的量值。 有五種交易隔離等級：讀取未認可、讀取認可、可重複的讀取、可序列化和版本控制。  
  
 **translator DLL**  
 用來將資料從一個字元集轉譯至另一個字元集的 DLL。  
  
 **translator 安裝程式 DLL**  
 包含 translator 專屬安裝和設定功能的 DLL。  
  
 **兩階段認可**  
 以兩個階段認可分散式交易的流程。 在第一個階段中，交易處理器會檢查是否可以認可交易的所有部分。 在第二個階段中，會認可交易的所有部分。 如果交易的任何部分在第一個階段中指出無法認可，則不會發生第二個階段。 ODBC 不支援兩階段認可。  
  
 **類型指標**  
 傳遞至或從 ODBC 函數傳回的整數值，以指出應用程式變數、參數或資料行的資料類型。 ODBC 會定義 C 和 SQL 資料類型的類型指標。  
  
## <a name="v"></a>V  
 **檢視**  
 查看一或多個資料表中資料的另一種方式。 視圖通常會建立為一或多個資料表中的資料行子集。 在 ODBC 中，views 通常相當於資料表。
