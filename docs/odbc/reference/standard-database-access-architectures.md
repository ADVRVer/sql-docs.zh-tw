# <a name="standard-database-access-architectures"></a>標準資料庫存取架構
在查看上一節所述的資料庫存取元件時，將會發現其中兩個程式設計介面和資料流程通訊協定，是理想的標準化候選項目。 另外兩個元件（IPC 機制和網路通訊協定）不僅位於層級太低，還會高度依賴網路和作業系統。 另外還有第三種方法-閘道，可提供標準化的可能性。  
  
 此章節包含下列主題。  
  
-   [標準程式設計介面](../../odbc/reference/standard-programming-interface.md)  
  
-   [標準資料流通訊協定](../../odbc/reference/standard-data-stream-protocol.md)  
  
-   [標準閘道](../../odbc/reference/standard-gateway.md)
