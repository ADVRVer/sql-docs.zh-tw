# <a name="using-block-cursors"></a>使用區塊資料指標
內建的區塊資料指標支援 ODBC 3。*x*。 **SQLFetch**所以只可用於與多資料列提取呼叫在 ODBC 3 時。*x*; 如果 ODBC 2。*x*應用程式會呼叫**SQLFetch**，它會開啟僅有單一資料列、 順向資料指標。 當 ODBC 3。*x*應用程式會呼叫**SQLFetch** ODBC 2。*x*驅動程式，它會傳回單一資料列的驅動程式支援，除非**SQLExtendedFetch**。 如需詳細資訊，請參閱 <<c0> [ 區塊資料指標、 可捲動的資料指標和回溯相容性](../../../odbc/reference/appendixes/block-cursors-scrollable-cursors-and-backward-compatibility.md)附錄 g： 驅動程式指導方針，為了與舊版相容。  
  
 若要使用區塊資料指標，應用程式設定的資料列集大小、 繫結資料列集的緩衝區 （如前一節所述），選擇性地設定 SQL_ATTR_ROWS_FETCHED_PTR 和 sql_attr_row_status_ptr 設定陳述式屬性，並呼叫**SQLFetch**或是**SQLFetchScroll**來提取資料列的區塊。 應用程式可以變更資料列集大小，並將新的資料列集緩衝區繫結 (藉由呼叫**SQLBindCol**或指定的繫結位移) 甚至已經提取資料列之後。
