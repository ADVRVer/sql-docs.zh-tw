# <a name="choosing-a-level-of-interoperability"></a>選擇互通性層的級
假設應用程式會使用 ODBC 下, 一個步驟是決定需要何種互通性層級。 互通性的基本層級通常是應用程式類型的函式： 自訂應用程式通常不是互通性最佳、 垂直應用程式，通常是有限數目的 Dbms，之間的互通性和泛型應用程式通常無法互通在所有的 Dbms。  
  
 此章節包含下列主題。  
  
-   [自訂應用程式](../../../odbc/reference/develop-app/custom-applications.md)  
  
-   [垂直應用市場](../../../odbc/reference/develop-app/vertical-applications.md)  
  
-   [泛型應用程式](../../../odbc/reference/develop-app/generic-applications.md)
