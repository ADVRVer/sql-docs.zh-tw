# <a name="events-generated-by-the-odbc-driver-manager"></a>ODBC 驅動程式管理員所產生的事件
> [!IMPORTANT]  
>  從 Windows 8 開始刪除了對可視化工作室分析器的支援(視覺工作室分析器僅包含在舊版本的 Visual Studio 中)。 對於其他故障排除機制,請使用 BID 跟蹤。  
  
 按下「開始視覺化工作室分析器」按鈕時,將註冊由ODBC驅動程式管理器生成的事件。 該工具本身提供系統定義的事件和創建自定義事件的能力。 有關事件的詳細資訊,請參閱可視化工作室文檔套件中的*視覺化工作室分析器參考指南*。  
  
|視覺化工作室分析器事件|描述|  
|----------------------------------|-----------------|  
|**呼叫**|在每個 ODBC API 條目上生成。|  
|**返回例外**|如果返回代碼SQL_ERROR,則在每個 ODBC API 返回上生成。|  
|**返回法線**|如果未SQL_ERROR返回代碼,則在每個 ODBC API 上生成。|  
|**連線啟動**|指示連接已啟動;當 ODBC 驅動程式管理器呼叫驅動程式的連接 API 時生成。|  
|**連線完成**|指示連接已完成;當驅動程式的連接 API 傳回到 ODBC 驅動程式管理器時生成。|  
|**斷線開始**|當 ODBC 驅動程式管理器呼叫驅動程式的**SQLDisconnect**函數時生成。|  
|**斷線完成**|當驅動程式的**SQLDisconnect**函數傳回到 ODBC 驅動程式管理器時生成。|  
|**查詢傳送**|當 ODBC 驅動程式管理器呼叫驅動程式的 SQLPrepare、SQLExecute、SQLExecDirect 函數以及目錄函數(如**SQLTables**和**SQLColumns)** 時產生。 **SQLPrepare** **SQLExecute** **SQLExecDirect**|  
|**查詢結果**|當驅動程式將結果集返回到涉及查詢的函數時生成。|  
|**交易開始**|當應用程式將SQL_ATTR_AUTOCOMMIT的值設置為SQL_AUTOCOMMIT_OFF時,或在應用程式成功調用**SQLEndTran**之後生成。|  
|**交易提交**|當應用程式調用**SQLEndTran**提交本地事務時生成。|  
|**交易回滾**|當應用程式調用**SQLEndTran**回滾本地事務時生成。|  
|**加入DTC**|當應用程式加入分散式事務協調器 (DTC) 時生成。|  
|**離開DTC**|當應用程式離開分散式事務協調器 (DTC) 時產生。|
