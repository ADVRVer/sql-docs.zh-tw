# <a name="types-of-descriptors"></a>描述項的類型
描述符用於描述以下描述之一:  
  
-   一組零個或多個參數。 參數描述符可用於描述:  
  
    -   *應用程式參數緩衝區,* 它包含應用程式設定的輸入動態參數或 SQL 的**CALL**語句執行後的輸出動態參數。  
  
    -   *實現參數緩衝區*。 對於輸入動態參數,它包含與應用程式參數緩衝區相同的參數,在應用程式可以指定的任何數據轉換之後。 對於輸出動態參數,這包含返回的參數,在應用程式可以指定的任何數據轉換之前。  
  
     對於輸入動態參數,應用程式在執行包含動態參數標記的任何 SQL 語句之前,必須對應用程式參數描述符進行操作。 對於輸入和輸出動態參數,應用程式可以指定與實現參數描述符中的數據類型不同的數據類型,以實現資料轉換。  
  
-   一行資料庫數據。 行描述符可用於描述:  
  
    -   *實作行緩衝區,* 其中包含資料庫中的行。 (這些緩衝區在概念上包含寫入資料庫或從資料庫讀取的數據。 但是,未指定資料庫數據的存儲形式。 資料庫可以在實現緩衝區中對其形式中的數據執行其他轉換。  
  
    -   *應用程式行緩衝區,* 其中包含向應用程式顯示的資料列,在應用程式可能指定的任何資料轉換之後。  
  
     應用程式在應用程式行描述符上運行,在任何情況下,資料庫中的列數據必須出現在應用程式變數中。 為了實現列資料的數據轉換,應用程式可以指定與實現行描述符中的數據類型不同的數據類型。  
  
 描述符類型在下表中匯總。  
  
|緩衝區型別|資料列|動態參數|  
|-----------------|----------|------------------------|  
|**應用程式緩衝區**|應用程式行描述符 (ARD)|應用程式參數描述符號 (APD)|  
|**實現緩衝區**|實作行描述符號 (IRD)|設定參數描述子 (IPD)|  
  
 對於參數或行緩衝區,如果應用程式在實現和應用程式描述符的相應記錄中指定不同的數據類型,則驅動程式在使用描述符時執行數據轉換。 例如,它可以將數值和日期時間值轉換為字串格式。 (有關有效的轉換,請參閱[附錄 D:資料類型](../../../odbc/reference/appendixes/appendix-d-data-types.md).)  
  
 描述符可以執行不同的角色。 不同的語句可以共用應用程式顯式分配的任何描述符。 一個語句中的行描述符可以用作另一個語句中的參數描述符。  
  
 始終知道給定描述符是應用程式描述符還是實現描述符,即使描述符尚未在資料庫操作中使用也是如此。 對於實現隱式分配的描述符,實現記錄相對於語句句柄的預定義行。 應用程式通過調用**SQLAllocHandle**分配的任何描述符都是應用程式描述符。
