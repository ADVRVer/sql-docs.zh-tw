# <a name="appendix-a-odbc-error-codes"></a>附錄 A：ODBC 錯誤碼
本主題討論 ODBC 3 的 SQLSTATE 值。*x*。 如需 ODBC 3 的詳細資訊，*x* SQLSTATE 值，請參閱 [SQLSTATE](../../../odbc/reference/develop-app/sqlstate-mappings.md)對應。  
  
 **SQLGetDiagRec** 或 **SQLGETDIAGFIELD** 會傳回 SQLSTATE 值，如 Open Group *資料管理：結構化查詢語言 (SQL)  (SQL) ，第2版* (1995 年3月) 。 SQLSTATE 值是包含五個字元的字串。 下表列出驅動程式可針對 **SQLGetDiagRec**傳回的 SQLSTATE 值。  
  
 針對 SQLSTATE 傳回的字元字串值包含兩個字元的類別值，後面接著三個字元的子類別值。 "01" 類別的值表示警告，並伴隨 SQL_SUCCESS_WITH_INFO 的傳回碼。 "01" 以外的類別值，除了類別 "IM" 之外，還表示錯誤，且伴隨著 SQL_ERROR 的傳回值。 「IM」類別是從 ODBC 本身的實作為衍生的警告和錯誤所特有。 任何類別中的子類別值 "000" 指出沒有該 SQLSTATE 的子類別。 類別和子類別值的指派是由 SQL-92 所定義。  
  
> [!NOTE]  
>  雖然函式的成功執行通常是以 SQL_SUCCESS 的傳回值來表示，但 SQLSTATE 00000 也表示成功。
