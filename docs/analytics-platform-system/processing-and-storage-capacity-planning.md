# <a name="processing-and-storage-capacity-in-analytics-platform-system"></a>分析平臺系統中的處理和儲存容量
您的商務需求會決定資料縮放單位的數目，以及您在 Analytics Platform System (AP) 設備所需的計算節點磁片大小。 使用這些處理和儲存體計算來引導您的容量購買和規劃決策。  
  
  
## <a name="planning-for-processing-capacity"></a><a name="section1"></a>規劃處理容量  
SQL Server 平行處理資料倉儲 (PDW) 的查詢效能，主要取決於平行處理資料的 CPU 核心數目。 在限制範圍內，增加平行處理原則可改善 (MPP) 查詢效能的大量平行處理。 即使您的資料大小相對較小，MPP 查詢引擎的威力也會藉由擁有較大的平行處理功能來增強。  
  
例如，具有12個計算節點的設備具有192個 CPU 核心，以平行方式處理您的資料。 這就是192方向的平行處理原則！ 具有56計算節點的設備具有896個核心，全都平行運作。 沒有 MPP 運算，就無法實現這種平行處理原則的程度。  
  
當計算節點數目增加時，相應放大設備需要一次新增一個以上的計算節點，以獲得顯著的好處。 硬體廠商僅支援資料縮放單位的特定設定，以確保調整設備的優點超過在更多計算節點上轉散發資料的成本。  
  
### <a name="data-scale-unit-configuration-examples---hpe"></a>資料縮放單位設定範例-HPE  
這些是資料縮放 Uunits 支援的 HPE 設定範例。 它們可能與最新支援的設定不同，但提供的範例是如何增加大約20% 的容量。  
  
升級是藉由將資料 Uunits 從一個資料列增加到下一個資料列來提高容量的百分比。 例如，將資料縮放單位從6增加為8，可在 CPU 核心和記憶體中提供33% 的升級。  它也會增加下表未顯示的磁碟空間。

-   每個設備的**記憶體**。 每個核心都有 256 GB 的記憶體。

## <a name="planning-for-storage-capacity"></a><a name="section2"></a>規劃儲存體容量  
下表估計您最多可以將 6 pb 的未壓縮資料載入並儲存到完整的分析平臺系統裝置上。 
  
|廠商|磁碟機大小|每個計算節點的實體資料儲存體|每個機架的計算節點數上限|每個機架的實體最大資料儲存空間|每個機架的估計使用者資料儲存空間上限|最大機架|每個設備的預估使用者資料儲存空間上限|  
|----------|--------------|------------------------------------------|----------------------------------|------------------------------------------|------------------------------------------------|-----------------|-----------------------------------------------------|  
|HPE|1 TB|16 TB|8|128 TB|320 TB|7|2240 TB|  
|HPE|2 TB|32 TB|8|256 TB|640 TB|7|4480 TB|  
|HPE|4 TB|64 TB|8|512 TB|1280 TB|7|8960 TB|  
|DELL|1 TB|16 TB|9|144 TB|360 TB|6|2160 TB|  
|DELL|2 TB|32 TB|9|288 TB|720 TB|6|4320 TB|  
|DELL|4 TB|64 TB|9|576 TB|1440 TB|6|8640 TB|   
  
解釋：  
  
-   每個硬體廠商的**磁片磁碟機大小**為1、2或 4 TB。  
  
-   **每個計算節點的實體資料儲存體** = (磁片磁碟機大小) * (每個計算節點的16個磁片) 。 鏡像磁片不包含在內，因為它們是用來進行冗余。  
  
-   **每個機架的計算節點數上限** 是硬體廠商專用的。  
  
-   **每個機架的實體最大資料儲存空間** = 每個計算節點 (實體資料儲存體) * (每個機架的計算節點數上限) 。  
  
-   **預估每個機架的使用者資料儲存空間上限** = (實體的每個機架最大資料儲存空間) * (5 表示5:1 壓縮比例) \* (50% 的記錄和 tempDB) 。 這是可載入並儲存在設備上的未壓縮使用者資料的保守估計。 這是預估值，並不會由軟體強制執行。 實際的使用者資料儲存體取決於您的資料和設定。  
  
-   **最大機架** 為每個硬體廠商所特有。  
  
-   **每個設備的預估資料儲存空間上限** = (預估每個機架的資料儲存空間上限) * (最大機架) 。 這是您可以載入並儲存在完全建立設備上的使用者資料大小總計的保守估計。
