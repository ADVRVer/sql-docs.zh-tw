# <a name="hardware-installation-for-analytics-platform-system-appliance"></a>Analytics Platform System appliance 的硬體安裝
本文說明如何移動、 打開包裝、 及安裝您的 SQL Server PDW 應用裝置的硬體。 本文章只是參考，並可協助您了解此程序。 應該打開包裝後，安裝，而且之前已給您驗證您的應用裝置。 客戶參與是必要項目，例如資料中心存取、 電力和乙太網路連線。  
  
## <a name="BeforeMoving"></a>在您從裝貨碼頭移動任何元件之前  
移動、 打開包裝、 或機架任何設備的元件之前，請執行下列工作。  
  
|工作|描述|  
|--------|---------------|  
|請確認所有元件已都到達|若要確認已到達的所有元件，以及位於您資料中心接收的停駐在其棧使用帳單的用料表 (BOM)。|  
|確認資料中心符合所有需求，應用裝置|開始這項工作，藉由檢閱硬體規格和纜線連接您 IHV 提供的圖表。 接下來的步驟提供的特定資訊機架空間和連線需求。|  
|確認資料中心有適當的機架空間|確認資料中心具有足夠空間可供所有設備的機架。<br /><br />確認機架空間是空的而且準備好接收在設備的機架。|  
|確認資料中心符合連線需求|確認資料中心符合纜線圖表中的纜線需求。<br /><br />確認會有所有的纜線和電源線的空間 racked 應用裝置節點之後。|  
|確認停駐和機架之間樓層符合權數需求|請確認所有地板拖板和機架之間可都支援應用裝置節點的加權，尤其是在資料中心的地板。<br /><br />如需每個元件的加權，請連絡 ihv 提供。|  
|安全資料中心的機架|保護資料中心就地使用您資料中心位置，例如地震輔助裝置，容易地震的地理區域中視需要額外的設備的機架。|  
|準備傳輸元件的協助|事先判斷哪些協助、 設備和工具，您必須處理每個元件，安全地而不會造成損害。|  
  
## <a name="Moving"></a>從裝貨碼頭中進入資料中心的機架  
每一組包含一個設備的機架，包括節點、 纜線、 電線等的所有元件。  
  
您可以使用下列檢查清單，會將每個設備的機架從裝貨碼頭在棧板中的資料中心的機架位置。 首先，移動控制項機架，然後移動 其他設備資料機架。  
  
> [!WARNING]  
> 請確實遵循說明執行這些步驟的失敗可能導致 bodily 傷害，您的 SQL Server PDW 應用裝置或其他問題所造成的損害。  
>   
> 永遠不會嘗試以提起或移動的應用裝置節點或其他大量的元件，而不需要協助或合適的設備。 連絡 ihv 提供了每個元件的加權，好讓您在何種協助、 設備和工具，您必須處理每個元件，而不會造成損毀並安全地可以事先判斷。  
  
|工作|描述|  
|--------|---------------|  
|確認這個棧板層級|開始移動或解除封裝這個棧板之前，先確定它位於層級的地面。|  
|Unbolt 節點從這個棧板|從這個棧板的頂端，unbolt 從這個棧板的最上層節點。|  
|將節點移到 dolly 或可支援加權的購物車|使用坡道和適當能拿起/移動技術來將節點移到 dolly 或可支援加權的購物車。|  
|在資料中心的傳輸 節點|您可以使用適當的能拿起/移動技術，將節點移到資料中心的機架中的位置。|  
|安全資料中心內的節點機架|保護資料中心內的位置中的節點機架。|  
|下一個節點或元件中重複這些步驟|重複上述步驟，將下一個節點或其他應用裝置元件移動到資料中心。|  
  
## <a name="AfterMoving"></a>安裝其他元件  
您可以使用下列檢查清單來安裝其他元件。  
  
|工作|描述||  
|--------|---------------|-|  
|打開包裝和網路交換器與 Pdu 機架|您可以使用 機架圖表來將網路交換器和 Pdu 放在適當的位置，在機架中。||  
|連接纜線標籤根據 Infiniband 和乙太網路纜線|請參閱纜線的圖表。 每個纜線兩端，指定連線需要有標籤。||  
|連接所有的電源纜線|請參閱纜線的圖表。||  
|開啟至機架和 Pdu 的電源供應器|電源供應器機架和從連接到 Pdu 機架。 **不開機任何其他的應用裝置元件的這一次。**||
