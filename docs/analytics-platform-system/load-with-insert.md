# <a name="load-data-with-insert-into-parallel-data-warehouse"></a>使用 INSERT into 平行處理資料倉儲載入資料

您可以使用 tsql INSERT 陳述式將資料載入 SQL Server Parallel Data Warehouse (PDW) 分散式或複寫資料表。 如需有關插入的詳細資訊，請參閱[插入](../t-sql/statements/insert-transact-sql.md)。 如需複寫的資料表和分散式資料表中的所有非散發資料行，PDW 會使用 SQL Server，將會隱含地轉換至目的地資料行的資料類型的陳述式中指定的資料值。 如需 SQL Server 資料轉換規則的詳細資訊，請參閱[資料類型轉換為 SQL](../t-sql/data-types/data-type-conversion-database-engine.md)。 不過，對於散發資料行，PDW 僅支援部分的 SQL Server 支援的隱含轉換。 因此，當您使用 INSERT 陳述式將資料載入至散發資料行時，來源資料必須指定其中一個下表中定義的格式。  
  
  
## <a name="InsertingLiteralsBinary"></a>插入的二進位類型的常值  
下表定義可接受的常值類型、 格式和常值插入散發資料行的類型轉換規則**二進位**(*n*) 或**varbinary**(*n*)。  
  
|常值類型|[格式]|轉換規則|  
|----------------|----------|--------------------|  
|二進位常值|0x*hexidecimal_string*<br /><br />範例： 0x12Ef|二進位常值必須加 0x。<br /><br />資料來源的長度不能超過指定的資料類型的位元組數目。<br /><br />如果資料來源長度的大小少於**二進位**資料類型，就會進行填補右側以零觸達的資料類型大小。|  
  
## <a name="InsertingLiteralsDateTime"></a>插入日期和時間類型的常值  
日期和時間常值會表示在特定的格式，括在單引號中使用字元值。 下表定義允許的常值類型、 格式和插入類型的 SQL Server PDW 散發資料行中的日期或時間常值的轉換規則**datetime**， **smalldatetime**，**日期**，**時間**， **datetimeoffset**，或**datetime2**。  
  
### <a name="datetime-data-type"></a>datetime 資料類型  
下表定義的可接受的格式和常值插入散發資料行類型的規則**datetime**。 任何空字串 （"） 轉換的預設值為 ' 1900年-01-01 12:00:00.000'。 字串，包含只有空白 (' ') 會產生錯誤。  
  
|常值類型|[格式]|轉換規則|  
|----------------|----------|--------------------|  
|字串常值**datetime**格式|' YYYY 為 YYYY-MM-DD hh: mm: [.nnn]'<br /><br />範例: ' 2007年-05-08 12:35:29.123'|插入值時，遺失的小數位數會設定為 0。 例如，常值 '2007年-05-08 12:35' 會插入做為 ' 2007年-05-08 12:35:00.000'。|  
|字串常值**smalldatetime**格式|' YYYY 為 YYYY-MM-DD hh: mm '<br /><br />範例: ' 2007年-05-08 12:35 '|秒數和剩餘的小數位數會設定為 0 時則會插入值。|  
|字串常值**日期**格式|'YYYY-MM-DD'<br /><br />範例: ' 2007年-05-08'|時間值 （小時、 分鐘、 秒和分數） 會設為 12:00:00.000 時則會插入值。|  
|字串常值**datetime2**格式|' YYYY-MM-DD 的-ss.nnnnnnn '<br /><br />範例: ' 2007年-05-08 12:35:29.1234567'|來源資料不能超過三個小數位數。 例如，常值 '2007年-05-08 12:35:29.123' 將會插入，但值' 2007年-05-08 12:35:29.1234567' 會產生錯誤。|  
  
### <a name="smalldatetime-data-type"></a>smalldatetime 資料類型  
下表定義的可接受的格式和常值插入散發資料行類型的規則**smalldatetime**。 任何空字串 （"） 轉換的預設值為 ' 1900年-01-01 12:00 '。 字串，包含只有空白 (' ') 會產生錯誤。  
  
|常值類型|[格式]|轉換規則|  
|----------------|----------|--------------------|  
|字串常值**smalldatetime**格式|'YYYY 為 YYYY-MM-DD hh: mm' 或者 'YYYY-MM-DD 的-hh:mm:00'<br /><br />範例: '2007年-05-08 12:00' 或 ' 2007年-05-08 12:00:00 '|來源資料必須具有值的年份、 月份、 日期、 小時和分鐘。 秒數是選擇性，而且如果有的話，必須設定為 00 的值。 任何其他值會產生錯誤。|  
|字串常值**日期**格式|'YYYY-MM-DD'<br /><br />範例: ' 2007年-05-08'|時間值 （小時、 分鐘、 秒和分數） 會設定為 0，則會插入值時。|  
  
### <a name="date-data-type"></a>Date 資料類型  
下表定義的可接受的格式和常值插入散發資料行類型的規則**日期**。 任何空字串 （"） 轉換的預設值為 ' 1900年-01-01'。 字串，包含只有空白 (' ') 會產生錯誤。  
  
|常值類型|[格式]|轉換規則|  
|----------------|----------|--------------------|  
|字串常值**日期**格式|'YYYY-MM-DD'<br /><br />範例: ' 2007年-05-08'|這是唯一接受的格式。|  
  
### <a name="time-data-type"></a>時間資料類型  
下表定義的可接受的格式和常值插入散發資料行類型的規則**時間**。 任何空字串 （"） 會轉換成預設值 '00:00:00.0000'。 字串，包含只有空白 (' ') 會產生錯誤。  
  
|常值類型|[格式]|轉換規則|  
|----------------|----------|--------------------|  
|字串常值**時間**格式|'hh:mm:ss.nnnnnnn'<br /><br />範例: ' 12:35:29.1234567'|如果資料來源具有小於或等於有效位數 （小數位數） 的有效位數比**時間**資料類型，資料會填補到右邊的零。 例如，常值 '12:35:29.123' 插入為 '12:35:29.1230000'。<br /><br />會拒絕具有較大的有效位數比目標資料類型的值。|  
  
### <a name="datetimeoffset-data-type"></a>datetimeoffset 資料類型  
下表定義的可接受的格式和常值插入散發資料行類型的規則**datetimeoffset** (*n*)。 預設的格式是 ' YYYY-MM-DD 的-ss.nnnnnnn {+ |-} hh: mm '。 空字串 （"） 轉換的預設值為 ' 1900年-01-01 12:00:00.0000000 + 00:00 '。 字串，包含只有空白 (' ') 會產生錯誤。 小數位數的數目取決於資料行定義。 例如，定義為資料行**datetimeoffset** (2) 將會有兩個小數位數。  
  
|常值類型|[格式]|轉換規則|  
|----------------|----------|--------------------|  
|字串常值**datetime**格式|' YYYY 為 YYYY-MM-DD hh: mm: [.nnn]'<br /><br />範例: ' 2007年-05-08 12:35:29.123'|遺漏的小數位數和位移的值會設定為 0 時則會插入值。 例如，常值 '2007年-05-08 12:35:29.123' 會插入做為' 2007年-05-08 12:35:29.1230000 + 00:00 '。|  
|字串常值**smalldatetime**格式|' YYYY 為 YYYY-MM-DD hh: mm '<br /><br />範例: ' 2007年-05-08 12:35 '|秒數、 剩餘的小數位數和位移的值會設定為 0 時則會插入值。|  
|字串常值**日期**格式|'YYYY-MM-DD'<br /><br />範例: ' 2007年-05-08'|時間值 （小時、 分鐘、 秒和分數） 會設定為 0，則會插入值時。 例如，常值 '2007年-05-08' 會插入做為' 2007年-05-08 00:00:00.0000000 + 00:00 '。|  
|字串常值**datetime2**格式|' YYYY-MM-DD 的-ss.nnnnnnn '<br /><br />範例: ' 2007年-05-08 12:35:29.1234567'|來源資料不能超過指定的 datetimeoffset 資料行中的小數秒數。 如果資料來源有小於或等於的小數秒數，就會進行填補右側以零。 比方說，如果資料類型為 datetimeoffset (5)，常值 '2007年-05-08 12:35:29.123 + 12:15' 會插入做為 ' 12:35:29.12300 + 12:15 '。|  
|字串常值**datetimeoffset**格式|' YYYY-MM-DD 的-ss.nnnnnnn {+&#124;-} hh: mm '<br /><br />範例: ' 2007年-05-08 12:35:29.1234567 + 12:15 '|來源資料不能超過指定的 datetimeoffset 資料行中的小數秒數。 如果資料來源有小於或等於的小數秒數，就會進行填補右側以零。 比方說，如果資料類型為 datetimeoffset (5)，常值 '2007年-05-08 12:35:29.123 + 12:15' 會插入做為 ' 12:35:29.12300 + 12:15 '。|  
  
### <a name="datetime2-data-type"></a>datetime2 資料類型  
下表定義的可接受的格式和常值插入散發資料行類型的規則**datetime2** (*n*)。 預設格式為 'YYYY-MM-DD 的-ss.nnnnnnn'。 空字串 （"） 轉換的預設值為 ' 1900年-01-01 12:00:00 '。 字串，包含只有空白 (' ') 會產生錯誤。 小數位數的數目取決於資料行定義。 例如，定義為資料行**datetime2** (2) 將會有兩個小數位數。  
  
|常值類型|[格式]|轉換規則|  
|----------------|----------|--------------------|  
|字串常值**datetime**格式|' YYYY 為 YYYY-MM-DD hh: mm: [.nnn]'<br /><br />範例: ' 2007年-05-08 12:35:29.123'|小數秒數是選擇性的則會插入值時，會設定為 0。<br /><br />會拒絕具有比目標資料類型的多個小數位數的值。|  
|字串常值**smalldatetime**格式|' YYYY 為 YYYY-MM-DD hh: mm '<br /><br />範例: ' 2007年-05-08 12'|選擇性的秒數和剩餘的小數位數會設定為 0 時則會插入值。|  
|字串常值**日期**格式|'YYYY-MM-DD'<br /><br />範例: ' 2007年-05-08'|時間值 （小時、 分鐘、 秒和分數） 會設定為 0，則會插入值時。 例如，常值 '2007年-05-08' 會插入做為' 2007年-05-08 12:00:00.0000000'。|  
|字串常值**datetime2**格式|' YYYY-MM-DD 的-hh:mm:ss:nnnnnnn'<br /><br />範例: ' 2007年-05-08 12:35:29.1234567'|如果資料來源包含日期和時間會小於或等於指定值的元件**datetime2**(*n*)，將資料插入; 否則會產生錯誤。|  
  
## <a name="InsertLiteralsNumeric"></a>插入數值類型的常值  
下表定義可接受的格式和使用數值類型的 SQL Server PDW 散發資料行中插入常值的轉換規則。  
  
### <a name="bit-data-type"></a>bit 資料類型  
下表定義的可接受的格式和常值插入散發資料行類型的規則**元**。 空字串 （"） 或字串，包含只有空白 (' ') 轉換成 0。  
  
|常值類型|format|轉換規則|  
|----------------|----------|--------------------|  
|字串常值**整數**格式|' nnnnnnnnnn'<br /><br />範例: '1' 或者 '而 321'|格式化為字串常值的整數值不能包含負數值。 例如，'-123' 的值會產生錯誤。<br /><br />大於 1 的值會轉換成 1。 例如，'123' 的值會轉換成 1。|  
|字串常值|'TRUE' 或者 'FALSE'<br /><br />範例: '，則為 true'|值 'TRUE' 會轉換為 1。值 'FALSE' 會轉換成 0。|  
|整數常值|nnnnnnnn<br /><br />範例： 1 或而 321|大於 1 或小於 0 的值會轉換成 1。 比方說，123 和-123 的值會轉換為 1。|  
|十進位常值|nnnnn.nnnn<br /><br />範例： 1234.5678|大於 1 或小於 0 的值會轉換成 1。 例如 123.45 和-123.45 值會轉換為 1。|

|字串常值**十進位**格式|'nnnnnn.nnnnn'<br /><br />範例: ' 123344.34455'|  
|整數常值|nnnnnnnnnnnn<br /><br />範例： 321312313123|  
|十進位常值|nnnnnn.nnnnn<br /><br />範例: ' 123344.34455'|

|常值類型|[格式]|轉換規則|

|字串常值**十進位**格式|'nnnnnn.nnnnn'<br /><br />範例: ' 123344.34455'|如果小數點後數字數目超過 4，值將會無條件進位到最接近值。 例如，'123344.34455' 的值會插入為 123344.3446。|  
|字串常值**金錢**格式|' $nnnnnn.nnnn'<br /><br />範例: ' $ 123456.7890'|選擇附加貨幣符號不會插入的值。<br /><br />如果小數點後數字數目超過 4，值將會無條件進位到最接近值。|  
|整數常值|nnnnnnnn<br /><br />範例： 123433|插入值時，遺失的數字的小數點後會設定為 0。 例如，常值 12345 插入為 12345.0000。|  
|十進位常值|nnnnnn.nnnnn<br /><br />範例： 123344.34455|如果小數點後數字數目超過 4，值將會無條件進位到最接近值。 例如，為 123344.3446 插入值 123344.34455。|  
|Money 常值|$nnnnnn.nnnn<br /><br />範例: $ 123456.7890|選擇附加貨幣符號不會插入的值。<br /><br />如果小數點後數字數目超過 4，值將會無條件進位到最接近值。|

### <a name="char-varchar-nchar-and-nvarchar-data-types"></a>char、 varchar、 nchar 和 nvarchar 資料類型  
下表定義的可接受的格式和常值插入散發資料行類型的規則**char**， **varchar**， **nchar**和**nvarchar**。 資料來源的長度不能超過指定的資料類型的大小。 如果資料來源長度的大小少於**char**或是**nchar**資料類型，就會進行填補空格右邊觸達的資料類型大小。  
  
|常值類型|[格式]|轉換規則|  
|----------------|----------|--------------------|

|整數常值|格式： nnnnnnnnnnn<br /><br />範例： 321312313123| None |  
|十進位常值|格式： nnnnnn.nnnnnnn<br /><br />範例： 12344.34455| None |  
|Money 常值|格式： $nnnnnn.nnnnn<br /><br />範例: $ 123456.99|貨幣符號不會插入的值。 若要插入的貨幣符號，插入的字串常值的值。 這會比對的格式**dwloader**工具，每個常值視為字串常值。<br /><br />不允許使用逗號。<br /><br />如果小數點後數字數目超過 2，值將會無條件進位到最接近值。 例如，為 123.95 插入值 123.946789。<br /><br />使用 CONVERT 函式插入 money 常值時，將允許只有預設樣式 0 （沒有逗號和小數點後的 2 個數字）。|
