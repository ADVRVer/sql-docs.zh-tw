# <a name="ado-glossary"></a>ADO 詞彙
本主題定義 ADO 的相關詞彙。  
  
## <a name="a"></a>A  
 絕對 URL  
 完整的 URL，指定位於網際網路或內部網路上的資源位置。 另請參閱*url*和*相對 url*。  
  
 ActiveX 控制項  
 自我註冊的同進程 COM 元件，其在設計階段或執行時間通常會有一個視覺元素。 ActiveX 控制項也能夠與活動文檔容器進行通訊，例如 Microsoft Internet Explorer。  
  
 ADISAPI （Advanced Data 網際網路伺服器應用程式開發介面）  
 提供剖析、自動化控制、記錄集封送處理和 MIME 封裝的 ISAPI DLL。 ADISAPI 元件會透過 Internet Information Services （IIS）所提供的 API 來運作。 另請參閱*ISAPI*。  
  
 彙總函式 (aggregate function)  
 在查詢中，會使用資料表資料行中的所有資料列來計算值的函數（例如 COUNT、AVG 或 STDEV）。 在撰寫運算式和程式設計時，您可以使用 SQL 彙總函式（包括上述三個）和網域彙總函式來判斷各種統計資料。  
  
 alias  
 您在 SQL SELECT 語句中提供給資料行或運算式的替代名稱，通常較短或更有意義。 例如，BobSales 是下列 SELECT 語句中的別名： "Select wr-Sales as BobSales from SalesDB"。 別名可以用來動態指派資料行，以控制 DataControl 物件上的系結。  
  
 Apartment 執行緒  
 COM 執行緒模型，其中物件的所有呼叫都會出現在一個執行緒上。 在單元執行緒中，COM 會同步處理和封送處理呼叫。 另請參閱*COMmddefcom*。  
  
 非同步作業  
 將控制權傳回給呼叫程式，而不需要等候作業完成的作業。 作業完成之前，會繼續執行程式碼。 另請參閱*同步*作業。  
  
## <a name="b"></a>B  
 binding 專案  
 資料表中的欄位與變數之間的對應。 在 ADO Visual C++ 延伸模組中，**記錄集**欄位會對應至 c/c + + 變數。  
  
 特  
 一個數值，適用于與其他數值的位元速率比較，通常是在參數或傳回值中加上旗標選項。 通常這項比較是使用位邏輯運算子來完成，例如**和**和，**或**在**&** Visual Basic 中，在 c + + 中 **&#124;** 。  
  
 例如，ADO **FieldAttributeEnum**值可以用來做為位元遮罩，以決定欄位的屬性。 假設您想要判斷欄位是否可更新。 您可以在 Visual Basic 中使用下列運算式來測試此項：`Field.Attributes AND adFldUpdatable`  
  
 如果結果為 TRUE，則欄位會是可更新的。  
  
 書籤 (bookmark)  
 可在一組資料列中唯一識別資料列的標記，讓使用者可以快速地流覽至該資料列。  
  
 商務物件  
 執行一組已定義之作業的物件，例如資料驗證或商務規則邏輯。 商務物件通常位於仲介層。  
  
 商務規則  
 驗證編輯、登入驗證、資料庫查閱、原則和演算法轉換的組合，構成企業執行業務的方式。 也稱為*商務邏輯*。  
  
## <a name="c"></a>C  
 計算運算式  
 不是常數的運算式，但其值相依于其他值。 若要進行評估，計算運算式必須取得並計算來自其他來源（通常在其他欄位或資料列中）的值。  
  
 第三章  
 資料來源中某個範圍之資料列的參考。 在 ADO 中，章節通常是另一個**記錄集**的參考。  
  
 章節資料行可讓您定義*父子式*關聯性，其中*父系*是包含 [章節] 資料行的**記錄集**，而*子*系是章節所代表的**記錄集**。  
  
 章節-別名  
 別名，參考附加至父系的資料行。  
  
 字元集 (character set)  
 一組字元與其數值的對應。 例如，Unicode 是16位字元集，能夠將所有已知的字元編碼，並當做全球字元編碼標準使用。  
  
 child  
 階層式關聯性的相依端。 子系是階層式結構中的節點，其上方有另一個節點（較接近根）。 另請參閱*子別名*、*父子式關聯*性、*父系*。  
  
 child-alias  
 參考子系的別名。 另請參閱*alias*、 *child*。  
  
 CLSID （類別識別碼）  
 識別 COM 元件的通用唯一識別碼（UUID）。 每個 COM 元件在 Windows 登錄中都有其 CLSID，讓其他應用程式可以載入它。 另請參閱*ProgID*、 *COM*。  
  
 用戶端層  
 分散式系統的邏輯層，通常會向使用者呈現資料，並處理來自使用者的輸入，有時稱為「*前端*」。 通常用戶端層會根據輸入向伺服器要求資料，然後格式化並顯示結果。 另請參閱*中介層*、*資料來源層*、*分散式應用程式*。  
  
 COM （元件物件模型）  
 一種二進位標準，讓物件可以在網路環境中互通，不論其開發的語言或其所在的電腦。 以 COM 為基礎的技術包括 ActiveX 控制項、自動化和物件連結和內嵌（OLE）。 COM 允許物件將其功能公開給其他元件，並裝載應用程式。 它會定義物件本身的公開方式，以及此公開在進程和網路之間的運作方式。 COM 也會定義物件的生命週期。  
  
 COM 元件  
 二進位檔案（例如 .dll、.ocx 和一些 .exe 檔案），支援提供物件的 COM 標準。 這類檔案包含一或多個 class factory、COM 類別、登錄輸入機制、載入程式碼等的程式碼。  
  
 比較運算子  
 比較兩個運算式並傳回布林值的運算子。  
  
 可以表示為 ">" （大於）、"\<" （小於）、"=" （等於）、">=" （大於或等於）、"<=" （小於或等於）、"<>" （不等於）或 "like" （模式比對）的準則參數。  
  
 component  
 封裝資料和程式碼的物件，並提供一組妥善指定的公開可用服務。  
  
 複合檔案  
 檔案的 COM 結構化儲存區的執行。 複合檔案會將不同的物件儲存在單一結構化檔案中，其中包含兩個主要元素：儲存物件和資料流程物件。 它們會一起運作，就像檔案中的檔案系統一樣。  
  
 一個實體檔案中系結在一起的多個個別檔案。 複合檔案中的每個個別檔案都可以如同單一實體檔案一樣存取。  
  
 常數  
 不會變更的數值或字串值。 您可以在程式碼中使用命名的 ADO 列舉（列舉常數），而不是實際的值，例如， **adUseClient**是其值為3的常數。 （Const adUseClient = 3）。 另請參閱*列舉*。  
  
 cursor  
 資料庫元素，可控制記錄導覽、資料的可更新性，以及其他使用者對資料庫所做之變更的可見度。  
  
## <a name="d"></a>D  
 資料繫結  
 將應用程式的物件或控制項關聯至資料來源的進程。 與資料來源相關聯的控制項稱為*資料繫結控制項*。  
  
 資料繫結控制項的內容會與資料庫中的值相關聯。 例如，當**記錄集**內的資料列更新時，可以更新系結至**記錄集**物件的方格控制項。 當**記錄集**抓取新的值時，新的值會顯示在方格中。  
  
 資料提供者 (data provider)  
 直接或透過服務提供者公開資料給 ADO 應用程式的軟體。 另請參閱服務提供者。  
  
 資料成形  
 使用正規化語法（稱為「**圖形語言**」）來定義特定**記錄集**物件（稱為「形狀的*記錄集*」）的技術，其中不只包含資料，還會根據其他記錄**集**物件的參考，以及其他**記錄**集物件和/或計算的值。  
  
 資料來源層  
 分散式系統的邏輯層，代表執行 DBMS 的電腦，例如 SQL Server 資料庫。 另請參閱*用戶端層*、*中介層*、*分散式應用程式*。  
  
 DCOM  
 一種有線通訊協定，可讓 COM 元件透過網路直接彼此通訊。 另請參閱*COM*、*元件*。  
  
 DDL （資料定義語言）  
 SQL 中定義的語句，而不是運算元據。 使用 DDL 建立或修改資料庫的架構。 例如， **CREATE TABLE**、 **CREATE INDEX**、 **GRANT**和**REVOKE**都是 SQL DDL 語句。  
  
 預設資料流程  
 使用特定 OLE DB 提供者（例如，用於網際網路發行的 Microsoft OLE DB 提供者）時，與**記錄**或**記錄集**物件相關聯的文字或二進位資料流程（以**資料流程**物件表示）。 預設資料流程通常包含檔案的內容，例如網站根目錄的 HTML 程式碼。  
  
 分散式應用程式  
 撰寫程式，以便在網路上跨多部電腦分割處理。 一般來說，分散式應用程式會分成簡報、商務邏輯和資料存放區層，或*分層*。 另請參閱用戶端層、中介層、資料來源層。  
  
 中斷連接的記錄集  
 用戶端快取中的**記錄集**物件，不再具有伺服器的即時連接。 如果原始資料來源因某些原因而需要再次存取，例如更新資料，則必須重新建立連接。 不過，您仍然可以存取已中斷連接之**記錄集**的集合、屬性和方法。  
  
 DML （資料操作語言）  
 SQL 中的語句，可操作（而不是定義）資料。 系統會選取資料庫中的值，並使用 DML 加以修改。 例如， **INSERT**、 **UPDATE**、 **DELETE**和**SELECT**是 SQL DML 語句。  
  
 檔來源提供者  
 管理資料夾和檔的特殊提供者類別。 當檔由**記錄**物件表示，或檔的資料夾由**記錄集**物件表示時，檔來源提供者會使用一組唯一的欄位來填入這些物件，以描述檔的特性，而不是實際的檔本身。 另請參閱資源記錄。  
  
 DSN （資料來源名稱）  
 用來將應用程式連接到特定 ODBC 資料庫的資訊集合。 ODBC 驅動程式管理員會使用此資訊來建立與資料庫的連接。 DSN 可以儲存在檔案（檔案 DSN）或 Windows 登錄（電腦 DSN）中。  
  
 動態屬性  
 資料提供者或資料指標服務特有的屬性。 物件的 Properties 集合會自動填入這些**屬性**（「動態」）。 物件在透過特定資料提供者連接到資料來源之前，沒有動態屬性。 另請參閱資料提供者、資料指標。  
  
## <a name="e"></a>E  
 列舉型別  
 具名常數的清單。 列舉值不需要是唯一的。 不過，在定義列舉的範圍內，每個值的名稱都必須是唯一的。 在 ADO 中，列舉是用於數值參數和傳回值，以將意義新增至 ADO 程式碼，並讓開發人員不受數值的保護（這可能會從版本變更為版本）。 例如，若要開啟靜態**記錄集**，請使用**adOpenStatic**列舉值：`Recordset.Open ,,adOpenStatic`  
  
 也稱為*列舉常數*。 另請參閱*常數*。  
  
 event  
 物件所識別的動作，您可以撰寫程式碼來回應。 事件可以透過命令執行、交易完成、記錄集流覽和資料更新，在其他動作中產生。 另請參閱*事件處理常式*。  
  
 事件處理常式 (event handler)  
 事件處理常式是事件發生時所執行的程式碼。 另請參閱事件。  
  
## <a name="h"></a>H  
 handler (處理常式)  
 管理一般和較簡單條件或作業的常式，例如錯誤修復或資料管理。  
  
 階層式記錄集  
 包含另一個**記錄集**的**記錄集**。 另請參閱資料成形，一節。  
  
 如需詳細資訊，請參閱[存取階層式記錄集中](../../ado/guide/data/accessing-rows-in-a-hierarchical-recordset.md)的資料列。  
  
 階層  
 一般而言，階層是具有最上層和從屬層級的排名結構。 在 ADO 中，階層式**記錄集**是用來表示記錄和章節之間的父子關聯性。 此外，您也可以在 ADO 中使用**Record**和**Stream**物件來存取階層式樹狀結構，例如資料夾和檔。 ADO MD**也包含階層**物件，以代表 OLAP Cube 中維度層級之間的關聯性。 另請參閱階層式記錄集、父系-子系關聯性、章節、樹狀結構。  
  
## <a name="i-l"></a>I-L  
 ISAPI （網際網路伺服器應用程式開發介面）  
 一組適用于網際網路伺服器的功能，例如執行 Microsoft® Internet Information Services （IIS）的 Windows NT® Server/Windows 2000 伺服器。  
  
 Key  
 唯一識別資料列的資料表中的一個或多個資料行;通常用來編制資料表索引。  
  
## <a name="m"></a>M  
 marshaling  
 跨執行緒或進程界限封裝、傳送和 unpackaging 介面方法參數的程式。  
  
 中介層  
 分散式系統中使用者介面或 Web 用戶端與資料庫之間的邏輯層。 這通常是用來具現化商務物件的地方。 中介層是商務規則和函式的集合，可在接收資訊時產生和操作。 他們會透過可能經常變更的商務規則來完成此動作，因此會封裝成與應用程式邏輯本身實際分開的元件。 也稱為*應用程式伺服器層*。 另請參閱分散式應用程式、用戶端層、資料來源層。  
  
 MIME （多用途網際網路郵件延伸模組）  
 一種網際網路通訊協定，最初開發的目的是為了允許在不同的網路、電腦和電子郵件環境中，以豐富內容交換電子郵件訊息。 實際上，非郵件應用程式也已採用並延伸 MIME。  
  
 MIME 是一種標準，可在網際網路上發佈和讀取二進位資料。 具有二進位資料之檔案的標頭包含資料的 MIME 類型;這會通知用戶端程式（例如，網頁瀏覽器和郵件套件），他們需要以不同的方式處理資料，而非處理純文字。 例如，包含 JPEG 圖形的 Web 檔標頭包含 JPEG 檔案格式特定的 MIME 類型。 這可讓瀏覽器以其 JPEG 檢視器顯示檔案（如果有的話）。  
  
## <a name="n-o"></a>N-O  
 node  
 階層式樹狀結構中的元素。 節點可以是根，或另一個節點的子系。 節點也可以是多個子系的父系。 另請參閱階層、樹狀結構、根、子系、父系。  
  
 物件變數  
 包含對物件之參考的變數。 例如， `objCustomObject`是指向 CustomObject 類型物件的變數：`Set objCustomObject = CreateObject(adodb.Recordset)`  
  
 ODBC (開放式資料庫連接)  
 一種標準的程式設計語言介面，用來連接到各種不同的資料來源。 這通常是透過 [控制台] 來存取，其中的資料來源名稱（Dsn）可指派為使用特定的 ODBC 驅動程式。  
  
 OLE DB  
 一組介面，會使用 COM 公開各種來源的資料。 OLE DB 介面可讓應用程式以一致的方式存取儲存在多樣化資訊來源中的資料。 這些介面支援適用于資料來源的 DBMS 功能數量，讓它能夠共用其資料。 另請參閱 COM。  
  
 開放式鎖定  
 一種鎖定類型，其中包含一或多筆記錄的資料頁面（包括正在編輯的記錄）只有在**更新**方法更新記錄時才無法供其他使用者使用，但會在呼叫**更新**之前和之後取得。  
  
 當使用**LockType**參數或屬性設定為**adLockOptimistic**或**AdLockBatchOptimistic**時，開啟**記錄集**物件時，會使用開放式鎖定。 另請參閱封閉式鎖定。  
  
 序數值  
 訂單內專案的數值位置。 在 ADO 集合中，第一個專案的序數值為零（0）。 下一個專案是一（1），依此類推。  
  
## <a name="p"></a>P  
 參數化命令  
 查詢或命令，可讓您在執行命令之前設定參數值。 例如，您可以藉由在 SQL 字串（由 '？ ' 字元指定）中內嵌參數標記，將 SQL 字串參數化。 然後，應用程式會指定每個參數的值，並執行命令。  
  
 父系 (parent)  
 階層式關聯性的控制端。 在階層式結構中，父系的一個或多個子節點緊接在階層中。 另請參閱父系-別名、父子式關聯性、子系。  
  
 父系-別名  
 參考父系的別名。 另請參閱 alias、parent。  
  
 父子式關聯性  
 階層式結構中的關聯性，其中父系層級較高，且直接與一或多個子系相關聯。 子系層級較低，且必須有一個父系。 另請參閱父系、子系。  
  
 封閉式鎖定  
 一種鎖定類型，其中包含一或多筆記錄（包括正在編輯的記錄）的頁面無法供其他使用者使用，以確保會進行更新。 封閉式鎖定行為是由 OLE DB 提供者所定義。 一般來說，記錄會在編輯時鎖定，而且在**更新**方法完成之前仍無法使用。  
  
 當使用**LockType**參數或屬性設定為**adLockPessimistic**時，開啟**記錄集**物件時，會啟用封閉式鎖定。 另請參閱開放式鎖定。  
  
 共用  
 以使用預先配置的資源集合（例如物件或資料庫連接）為基礎的效能優化。 從集區中繪製現有資源比建立新資源更有效率。  
  
 ProgID （程式設計識別碼）  
 COM 應用程式對應至 Windows 登錄的唯一名稱。 ADO 連接的 ProgID 為 "ADODB"。連接」。 另請參閱 CLSID、COM。  
  
 proxy  
 介面特定的物件，提供用戶端呼叫在不同執行環境（例如在不同的執行緒或另一個進程）中執行之應用程式物件所需的參數封送處理和通訊。 Proxy 會與用戶端進行通訊，並與位於所呼叫應用程式物件的對應存根進行通訊。 另請參閱 stub。  
  
## <a name="r"></a>R  
 相對 URL  
 部分限定的 URL，指定網際網路或內部網路上的資源，其位置相對於絕對 URL 或對等 ADO 連線物件所指定的起始點。 實際上，串連的絕對和相對 Url 會 consitute 完整的 URL。 另請參閱 URL 和絕對 URL。  
  
 遠端資料源  
 存在於另一部電腦上的資料來源，而不是在本機系統上（用戶端應用程式執行所在的位置）。  
  
 資源記錄  
 來自檔來源提供者的記錄，其中包含資料夾或檔之定義和描述的欄位。 檔本身不會包含在資源記錄中，但通常可以透過預設資料流程或包含 URL 之資源記錄中的欄位來存取。 另請參閱檔來源提供者、預設資料流程、URL。  
  
 資料列集  
 來自資料來源的一組資料列，全都具有相同的欄位架構。 資料列集可以代表資料表中的所有或部分欄位。 資料列集也可以代表由查詢或聯結兩個或多個資料表所建立的虛擬資料表。 在 ADO 中，資料列集是由**Recordset**物件所表示。  
  
## <a name="s"></a>S  
 影響範圍  
 物件或變數的參考範圍，或是視圖或資料表中的記錄範圍。 例如，本機變數只能在其定義所在的程式內參考。 公用變數可從應用程式中的任何位置存取。 如目前資料庫的物件位於定義的搜尋路徑中，則會在範圍中。 您可以在許多命令中使用 Scope 子句來指定記錄範圍。  
  
 服務提供者  
 藉由產生和取用資料來封裝服務的軟體，擴充 ADO 應用程式中的功能。 它是不會直接公開資料的提供者，而是提供服務（例如查詢處理）。 服務提供者可能會處理資料提供者所提供的資料。 另請參閱資料提供者。  
  
 成形的記錄集  
 已明確定義資料行的**記錄集**，其中不只包含資料，也會參考（稱為章節）至其他**記錄**集物件和/或根據其他**記錄集**物件計算的值。  
  
 同層級  
 階層結構中相同層級上的兩個或多個節點。 階層中的根節點沒有任何同級。  
  
 預存程序  
 先行編譯的程式碼集合，例如 SQL 語句和選擇性的流程式控制制語句，儲存在名稱底下並當做一個單位來處理。 預存程式會儲存在資料庫內;您可以使用應用程式的一個呼叫來執行它們，並允許使用者宣告的變數、條件式執行，以及其他強大的程式設計功能。  
  
 存根  
 介面特定的物件，提供應用程式物件從在不同執行環境（例如在不同的執行緒或另一個進程）中執行之用戶端接收呼叫所需的參數封送處理和通訊。 存根位於應用程式物件中，並會與用戶端呼叫它的對應 proxy 進行通訊。 另請參閱 proxy。  
  
 子節點  
 請參閱子系。  
  
 同步作業  
 在下一個作業開始之前完成的程式碼所起始的作業。 另請參閱非同步作業。  
  
## <a name="t-z"></a>T-Z  
 樹狀結構  
 結構，代表元素（節點）之間的階層式關聯性。 樹狀結構的最上層（根）有一個節點。 在根底下，可以有多個子系。 每個子系可能會變成其他子系的父系，因此會像樹狀一樣進行分支。 包含檔和其他資料夾的資料夾是樹狀結構的典型範例。 另請參閱階層、節點、根、子系、父系。  
  
 網頁伺服器  
 提供 Web 服務和網頁給內部網路和網際網路使用者的電腦。
