# <a name="drop-server-audit--transact-sql"></a>DROP SERVER AUDIT  (Transact-SQL)
[!INCLUDE[tsql-appliesto-ss2008-xxxx-xxxx-xxx-md](../../includes/tsql-appliesto-ss2008-xxxx-xxxx-xxx-md.md)]

  使用 SQL Server Audit 功能卸除伺服器稽核物件。 如需 SQL Server Audit 的詳細資訊，請參閱 [SQL Server Audit &#40;Database Engine&#41;](../../relational-databases/security/auditing/sql-server-audit-database-engine.md)。  
  
 ![主題連結圖示](../../database-engine/configure-windows/media/topic-link.gif "主題連結圖示") [Transact-SQL 語法慣例](../../t-sql/language-elements/transact-sql-syntax-conventions-transact-sql.md)  
  
## <a name="syntax"></a>語法  
  
```  
  
DROP SERVER AUDIT audit_name  
    [ ; ]  
```  
  
## <a name="remarks"></a>Remarks  
 您必須將稽核的狀態設定為 OFF 選項，才能對稽核進行任何變更。 如果在設定 STATE=OFF 以外的任何選項時啟用稽核，而且執行 DROP AUDIT，您將會收到 MSG_NEED_AUDIT_DISABLED 錯誤訊息。  
  
 DROP SERVER AUDIT 會移除稽核的中繼資料，但不會移除命令發出前所收集的稽核資料。  
  
 DROP SERVER AUDIT 不會卸除關聯的伺服器或資料庫稽核規格。 這些規格必須手動卸除或維持遺棄的狀態，之後會對應到新的伺服器稽核。  
  
## <a name="permissions"></a>Permissions  
 若要建立、更改或卸除伺服器稽核，主體需要使用 ALTER ANY SERVER AUDIT 或 CONTROL SERVER 權限。  
  
## <a name="examples"></a>範例  
 下列範例會卸除稱為 `HIPAA_Audit` 的稽核。  
  
```  
ALTER SERVER AUDIT HIPAA_Audit  
STATE = OFF;  
GO  
DROP SERVER AUDIT HIPAA_Audit;  
GO  
```
