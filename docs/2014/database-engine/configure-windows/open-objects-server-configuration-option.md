# <a name="open-objects-server-configuration-option"></a>開啟物件伺服器組態選項
  雖然在 中已停用此選項的功能，但此選項仍存在於 [!INCLUDE[msCoName](../../includes/msconame-md.md)] [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)]. (此設定無效)。在 [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] 中，已開啟之資料庫物件的數目會受到動態管理，而且只受可用記憶體的限制。 **open objects** 選項可在 **sp_configure** 中使用，以保有與現有指令碼之間的回溯相容性。
